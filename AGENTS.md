# OpenMeter

## Make Targets

Use the `Makefile` for all common tasks. Dagger and justfile exist but are seldom used.

| Target | Purpose |
|--------|---------|
| `make test` | Run tests (requires `POSTGRES_HOST=127.0.0.1`) |
| `make test-nocache` | Run tests bypassing cache (`-count=1`) |
| `make test-all` | Full suite with external deps (Svix, Redis) |
| `make etoe` | E2E API tests via docker compose ŌĆö always use Make |
| `make generate` | Ent ORM + Wire DI + Goverter + Goderive |
| `make gen-api` | TypeSpec -> OpenAPI -> oapi-codegen -> Go + JS/Python SDKs |
| `make generate-all` | All codegen combined |
| `make lint` / `make lint-go` | Linting (golangci-lint) |
| `make fmt` | Format with golangci-lint --fix |
| `make up` / `make down` | Docker compose deps |
| `make server` | Dev server with hot reload (air) |

## Testing

Running a single package directly:
```bash
POSTGRES_HOST=127.0.0.1 go test -tags=dynamic -v ./openmeter/billing/...
```

Key flags: `-tags=dynamic` (required for confluent-kafka-go), `-p 128 -parallel 16` (used by Make).

Patterns:
- **testify** for assertions (`require.*` to fail fast, `assert.*` to continue)
- **pgtestdb** for isolated Postgres per test ŌĆö skips if `POSTGRES_HOST` unset
- **testutils** packages per domain (e.g. `openmeter/testutils`, `openmeter/streaming/testutils`)
- **clock.SetTime / clock.FreezeTime** for time-deterministic tests (always `defer clock.ResetTime()`)
- **eventbus.NewMock(t)** for event bus in tests
- **Suite pattern**: `suite.Suite` + `*require.Assertions` embedded, with `SetupSuite`/`TearDownSuite`

## Architecture

**Entry points**: `cmd/server`, `cmd/billing-worker`, `cmd/balance-worker`, `cmd/sink-worker`, `cmd/notification-service`, `cmd/jobs`

**Domain packages** live under `openmeter/`. Each follows a 4-layer pattern:

```
openmeter/<domain>/
  <domain>.go          # Entity types, interfaces, errors
  service/service.go   # Business logic (implements Service interface)
  adapter/adapter.go   # Ent DB persistence (implements Adapter interface)
  httpdriver/           # HTTP handlers + API mapping
```

### Conventions When Writing Code

**Constructors & Config:**
```go
type Config struct {
    Adapter   customer.Adapter
    Publisher eventbus.Publisher
}
func (c Config) Validate() error { ... }
func New(config Config) (*Service, error) { ... }
```

**Input types** ŌĆö all implement `.Validate() error`:
```go
type CreateCustomerInput struct {
    Namespace string
    CustomerMutate
}
```
Naming: `Create<Entity>Input`, `Update<Entity>Input`, `List<Entity>Input`.

**Errors** ŌĆö wrap generic types from `pkg/models`:
- `models.NewGenericValidationError()`, `models.NewGenericNotFoundError()`, `models.NewGenericConflictError()`
- Domain errors: provide `New<X>Error()` constructor + `Is<X>Error()` checker
- Validation issues: `models.NewValidationIssue(code, message)` for structured client-facing errors

**Multi-tenancy** ŌĆö every entity is namespaced. Extract via `namespaceDecoder.GetNamespace(ctx)`.

**HTTP handlers** ŌĆö use `httptransport.HandlerWithArgs[Req, Resp, Params]` with two phases: parse HTTP -> domain input, then call service -> map response.

**Transactions** ŌĆö `transaction.Run(ctx, repo, func() error {...})`. Repos implement `transaction.Driver`.

**Hooks** ŌĆö `models.ServiceHookRegistry[T]` for lifecycle events (PreUpdate, PostCreate, etc.).

**Logging** ŌĆö `*slog.Logger` everywhere. Use `logger.WarnContext(ctx, "msg", "key", val)`.

**DI** ŌĆö Google Wire. Providers in `app/common/`. Each binary composes its own `wire.go`.

## Key Dependencies

- **DB**: PostgreSQL (Ent ORM, Atlas migrations, pgx driver)
- **Analytics**: ClickHouse
- **Events**: Kafka (confluent-kafka-go) + Watermill
- **HTTP**: Chi router + oapi-codegen
- **Billing**: Stripe, GOBL (invoicing)
- **Webhooks**: Svix
- **Observability**: OpenTelemetry
- **Config**: Viper + Cobra
- **Utilities**: samber/lo
