# billing

<!-- archie:ai-start -->

> v3 HTTP handlers for customer-scoped billing: reading/writing the billing profile override, app-specific customer data (Stripe, CustomInvoicing, Sandbox), and Stripe checkout/portal sessions. Every mutation must guard against deleted customers.

## Patterns

**Deleted-customer guard in the decoder** — Mutation decoders call h.customerService.GetCustomer and return apierrors.NewGoneError when cus.IsDeleted() is true before building the domain input. (`if cus.IsDeleted() { return ..., apierrors.NewGoneError(ctx, errors.New("customer is deleted")) }`)
**App-type switch kept in sync across read/write** — Handlers reading/writing app-specific data switch on application.GetType() over app.AppTypeStripe/AppTypeCustomInvoicing/AppTypeSandbox, returning apierrors.NewInternalError for unknown types; get_billing.go and update_billing.go must stay in sync. (`switch application.GetType() { case app.AppTypeStripe: ... default: return resp, apierrors.NewInternalError(ctx, fmt.Errorf("unsupported app type")) }`)
**Goverter for v3-to-legacy enum conversions** — convert.go declares goverter variable-set FromAPI*/ToAPI* funcs with goverter:enum:map annotations (and goverter:enum:unknown @error); convert.gen.go is generated and never hand-edited. (`// goverter:enum:map BillingAppStripeCreateCheckoutSessionBillingAddressCollectionAuto CreateStripeCheckoutSessionBillingAddressCollectionAuto
FromAPIBillingAppStripeCreateCheckoutSessionBillingAddressCollection func(...) (apilegacy.CreateStripeCheckoutSessionBillingAddressCollection, error)`)
**Domain error encoder appended after generic** — error_encoder.go matches billing.NotFoundError->404, ValidationError->400, UpdateAfterDeleteError->409, AppError->400 and is always appended AFTER apierrors.GenericErrorEncoder(). (`httptransport.WithErrorEncoder(apierrors.GenericErrorEncoder()), httptransport.WithErrorEncoder(errorEncoder())`)
**Stripe app resolved via billing profile** — Checkout session creation resolves the Stripe AppID through h.billingService.ResolveStripeAppIDFromBillingProfile; portal session uses GetCustomerApp with an AppType filter — never hardcode app IDs. (`appId, err := h.billingService.ResolveStripeAppIDFromBillingProfile(ctx, namespace, customerId)`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `handler.go` | Handler interface + handler struct (billingService, customerService, stripeService) and New(). | handler.appService is declared but NOT set in New(); adding app.Service to New()'s signature breaks existing call sites. |
| `convert.go` | Source of truth for all v3-to-legacy and domain conversions; goverter annotations drive convert.gen.go. | Adding a TypeSpec enum variant without a matching goverter:enum:map causes a runtime 'unexpected enum element' error. |
| `convert.gen.go` | Generated by goverter from convert.go (//go:build !goverter). | DO NOT EDIT; overwritten on next go generate. |
| `error_encoder.go` | Package-local encoder mapping billing-specific error types to HTTP codes. | Must be appended AFTER GenericErrorEncoder; reversing order makes billing.ValidationError fall through to 500. |
| `update_billing.go` | Upserts the billing profile override AND app customer data; update_billing_app_data.go handles app-data-only updates. | Shares the app-type switch with get_billing.go/update_billing_app_data.go — keep all in sync when adding an app type. |

## Anti-Patterns

- Calling billing domain services without checking customer.IsDeleted() in the decoder
- Hand-editing convert.gen.go instead of updating convert.go annotations and regenerating
- Appending errorEncoder() before GenericErrorEncoder() (reverses precedence)
- Hardcoding a Stripe app ID instead of resolving via ResolveStripeAppIDFromBillingProfile
- Adding app-type handling in only one of get_billing.go / update_billing.go

## Decisions

- **Stripe checkout/portal session endpoints live alongside generic billing profile endpoints** — They share the customer-scoped path prefix and identical deleted-customer validation; collocating avoids duplicating that validation.
- **Goverter (with @error on unknown) drives v3-to-legacy enum conversion** — The Stripe checkout enum space is large (10+ enums); generated mapping gives compile-time safety and avoids manual drift.
