# apps

<!-- archie:ai-start -->

> v3 HTTP handlers for listing and getting installed billing apps (Stripe, Sandbox, CustomInvoicing), performing polymorphic type-assertion on app.App to map each concrete type onto the apiv3.BillingApp oneOf union.

## Patterns

**Polymorphic app mapping via type-assertion switch** — ToAPIBillingApp in convert.go switches on item.GetType(), type-asserts to the concrete app (appstripe.App, appsandbox.App, appcustominvoicing.App), builds the per-type wire struct, then calls the matching From* on api.BillingApp. (`billingApp := api.BillingApp{}; if err := billingApp.FromBillingAppStripe(billingAppStripe); err != nil { return billingApp, err }`)
**Goverter for struct mappings, hand-code for unions** — convert.go carries goverter:variables annotations generating convert.gen.go for simple mappings (PagePaginatedResponse, BillingAppCatalogItem, AppType enum); the polymorphic ToAPIBillingApp is hand-written and listed via goverter:extend. (`//go:generate go run github.com/jmattheis/goverter/cmd/goverter gen ./`)
**handler struct holds app.Service** — Unlike other handlers holding a domain-specific service, the apps handler holds the top-level app.Service registry. (`type handler struct { resolveNamespace func(context.Context) (string, error); appService app.Service; options []httptransport.HandlerOption }`)
**Compile-time interface check** — handler.go includes var _ Handler = (*handler)(nil) to catch missing method implementations at compile time. (`var _ Handler = (*handler)(nil)`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `convert.go` | Hand-written ToAPIBillingApp polymorphic switch + per-type helpers (toAPIBillingAppStripe/Sandbox/ExternalInvoicing) + goverter annotations for list/catalog/enum conversions. | goverter:enum:map annotations must stay in sync with app.AppType constants (else @error). A new app type needs a switch case AND a toAPIBillingApp<Type> helper. AppTypeCustomInvoicing maps to BillingAppTypeExternalInvoicing. |
| `convert.gen.go` | Generated by goverter — DO NOT EDIT. Regenerate with go generate ./... after changing convert.go annotations. | Manual edits are overwritten on the next generate; build tag is //go:build !goverter. |
| `handler.go` | Handler interface (ListApps, GetApp) + New() constructor with var _ Handler = (*handler)(nil) satisfaction check. | New app endpoints must be added to the Handler interface. |

## Anti-Patterns

- Adding a new app type without a case in the ToAPIBillingApp switch
- Editing convert.gen.go directly instead of regenerating
- Accessing app-specific fields without first type-asserting to the concrete app type
- Bypassing ToAPIBillingApp to inline-convert in get_app.go or list_app.go

## Decisions

- **BillingApp oneOf mapping done via app.App interface -> concrete assertion -> From* call** — The generated union requires the From* discriminator method; no generic field copy works across heterogeneous concrete types.
- **Goverter only for struct-level mappings; the discriminated union is hand-coded** — Goverter cannot handle interface type-switches or oneOf union encoding.

## Example: Add a new billing app type to the polymorphic mapping

```
// In convert.go ToAPIBillingApp switch:
case app.AppTypeNewProvider:
    newApp, ok := item.(appnewprovider.App)
    if !ok {
        return api.BillingApp{}, fmt.Errorf("expected new-provider app, got %T", item)
    }
    billingAppNew, err := toAPIBillingAppNewProvider(newApp.Meta)
    if err != nil {
        return api.BillingApp{}, fmt.Errorf("failed to map new-provider app to API: %w", err)
    }
    billingApp := api.BillingApp{}
    if err := billingApp.FromBillingAppNewProvider(billingAppNew); err != nil {
        return billingApp, err
    }
    return billingApp, nil
```
