# billingprofiles

<!-- archie:ai-start -->

> HTTP handlers for billing profile CRUD in the v3 API; bridges billing.Service (ProfileService) to HTTP with a domain-specific error encoder and goverter-generated conversions for workflow configs, tax config, and supplier parties.

## Patterns

**Domain-specific error encoder chained with GenericErrorEncoder** — Every operation in this package appends both apierrors.GenericErrorEncoder() AND the local errorEncoder() which maps billing.NotFoundError→404, billing.ValidationError→400, billing.UpdateAfterDeleteError→409. (`httptransport.WithErrorEncoder(apierrors.GenericErrorEncoder()), httptransport.WithErrorEncoder(errorEncoder())`)
**Goverter context params for namespace injection** — Goverter-generated FromAPICreateBillingProfileRequest accepts a string context param (namespace) which gets mapped to the Namespace field via goverter:context and NamespaceFromContext helper. (`// goverter:context namespace
// goverter:map Namespace | NamespaceFromContext
FromAPICreateBillingProfileRequest func(namespace string, req api.CreateBillingProfileRequest) (billing.CreateProfileInput, error)`)
**ProfileExpand: Apps set on read operations** — GetBillingProfile and ListBillingProfiles always set billing.ProfileExpand{Apps: true} so the app references are populated in the response. (`Expand: billing.ProfileExpand{Apps: true}`)
**error_encoder.go isolates billing error mapping** — A dedicated error_encoder.go file returns an encoder.ErrorEncoder closure that handles billing-specific error types, keeping operation files free of error type assertions. (`commonhttp.HandleErrorIfTypeMatches[billing.NotFoundError](ctx, http.StatusNotFound, err, w, billing.EncodeValidationIssues)`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `convert.go` | Goverter annotations + hand-written helpers (ToAPIBillingWorkflow, FromAPIBillingWorkflow, ToAPIBillingParty, FromAPIBillingParty, ToAPIBillingAppReference). FromAPIBillingWorkflow parses ISO duration strings and handles alignment discriminated union. | FromAPIBillingWorkflow uses period.Parse for duration strings and falls back to billing.DefaultWorkflowConfig; both collection alignment variants (subscription, anchored) must be handled. |
| `error_encoder.go` | Maps billing.NotFoundError, billing.ValidationError, billing.UpdateAfterDeleteError, billing.ValidationIssue, billing.AppError to HTTP status codes using commonhttp.HandleErrorIfTypeMatches. | Order matters: more specific types should appear before more general ones in the chain. |
| `convert.gen.go` | Generated by goverter — DO NOT EDIT. Contains FromAPICreateBillingProfileRequest, FromAPIUpsertBillingProfileRequest, ToAPIBillingProfile, ToAPIBillingProfiles, ToAPIBillingTaxConfig, FromAPIBillingTaxConfig. | Regenerate after any change to convert.go annotations. |
| `get.go` | GetBillingProfile with ProfileExpand{Apps: true}; returns apierrors.NewNotFoundError if profile is nil after service call. | Nil profile check after service call must return apierrors.NewNotFoundError, not a generic error. |

## Anti-Patterns

- Omitting errorEncoder() from any billing profile operation — GenericErrorEncoder alone won't map billing.NotFoundError to 404
- Editing convert.gen.go manually
- Forgetting ProfileExpand{Apps: true} on read paths that need app reference data
- Adding new workflow fields to convert.go without updating both ToAPIBillingWorkflow and FromAPIBillingWorkflow

## Decisions

- **A local errorEncoder() is provided alongside GenericErrorEncoder because billing errors (NotFoundError, ValidationError, UpdateAfterDeleteError) are domain-specific types not handled by the generic encoder.** — The generic encoder handles models.Generic* errors; billing defines its own error types for richer validation issue propagation.
- **Goverter context params are used to thread namespace and namespacedID into generated converters instead of manual construction.** — Avoids boilerplate in every FromAPI function while keeping the generated code correct and auditable.
