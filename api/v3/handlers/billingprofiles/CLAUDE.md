# billingprofiles

<!-- archie:ai-start -->

> v3 HTTP handlers for billing profile CRUD; bridges billing.Service (ProfileService) to HTTP with a domain-specific error encoder and goverter-generated conversions for workflow configs, tax config, and supplier parties.

## Patterns

**Domain error encoder chained after GenericErrorEncoder** — Every operation appends both apierrors.GenericErrorEncoder() AND the local errorEncoder(); the local one must come last so it maps billing.NotFoundError->404, billing.ValidationError->400, billing.UpdateAfterDeleteError->409. (`httptransport.WithErrorEncoder(apierrors.GenericErrorEncoder()), httptransport.WithErrorEncoder(errorEncoder())`)
**Goverter context params for namespace/ID injection** — Generated FromAPI* funcs take a context param (namespace string or namespacedID models.NamespacedID) mapped via goverter:context + NamespaceFromContext/ResolveIDFromContext helpers instead of manual struct construction. (`// goverter:context namespace
// goverter:map Namespace | NamespaceFromContext
FromAPICreateBillingProfileRequest func(namespace string, req api.CreateBillingProfileRequest) (billing.CreateProfileInput, error)`)
**ProfileExpand{Apps:true} on read paths** — GetBillingProfile and ListBillingProfiles always set billing.ProfileExpand{Apps: true} so app references populate in the response. (`Expand: billing.ProfileExpand{Apps: true}`)
**TaxConfig dual-field wrappers** — toAPIBillingTaxConfigFull/fromAPIBillingTaxConfigFull wrap the goverter ToAPI/FromAPIBillingTaxConfig to keep new TaxCode and deprecated TaxCodeId fields in sync (TaxCode.Id takes precedence on input). (`if result.TaxCodeId != nil { result.TaxCode = &api.TaxCodeReference{Id: *result.TaxCodeId} }`)
**FromAPIBillingWorkflow defaults + discriminated alignment** — FromAPIBillingWorkflow starts from billing.DefaultWorkflowConfig, parses ISO durations via period.Parse with fallback, and handles both subscription and anchored collection-alignment union variants. (`if parsed, err := period.Parse(*workflow.Collection.Interval); err == nil { collInterval = datetime.ISODuration{Period: parsed} }`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `convert.go` | Goverter variable declarations + hand-written helpers (ToAPI/FromAPIBillingWorkflow, ToAPI/FromAPIBillingParty, ToAPIBillingAppReference) and TaxConfig dual-field wrappers. | FromAPIBillingWorkflow uses period.Parse and falls back to billing.DefaultWorkflowConfig; both alignment variants (subscription, anchored) must be handled. |
| `error_encoder.go` | Maps billing.NotFoundError/ValidationError/UpdateAfterDeleteError/ValidationIssue/AppError to HTTP status codes via commonhttp.HandleErrorIfTypeMatches. | Order matters: more specific types appear before more general ones; uses billing.EncodeValidationIssues for the validation-bearing types. |
| `convert.gen.go` | Generated by goverter (//go:build !goverter). Holds FromAPICreate/UpsertBillingProfileRequest, ToAPIBillingProfile(s), ToAPI/FromAPIBillingTaxConfig. | DO NOT EDIT; regenerate after any convert.go annotation change with go generate. |
| `get.go` | GetBillingProfile with ProfileExpand{Apps:true}; returns apierrors.NewNotFoundError when the service returns a nil profile. | Nil profile after the service call must return apierrors.NewNotFoundError, not a generic error. |
| `handler.go` | Handler interface (List/Create/Get/Update/Delete) and handler struct holding resolveNamespace closure, billing.Service, and options. | Uses the resolveNamespace func(ctx)(string,error) closure pattern; new ops must call h.resolveNamespace, not a namespace decoder. |

## Anti-Patterns

- Omitting errorEncoder() from a billing profile operation — GenericErrorEncoder alone won't map billing.NotFoundError to 404
- Editing convert.gen.go manually
- Forgetting ProfileExpand{Apps:true} on read paths needing app reference data
- Adding a new workflow field to convert.go without updating both ToAPIBillingWorkflow and FromAPIBillingWorkflow

## Decisions

- **A local errorEncoder() runs alongside GenericErrorEncoder** — Billing defines its own domain error types (NotFoundError, ValidationError, UpdateAfterDeleteError) that the generic models.Generic* encoder does not handle.
- **Goverter context params thread namespace/namespacedID into generated converters** — Avoids boilerplate in every FromAPI function while keeping the generated code auditable.
