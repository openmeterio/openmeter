# taxcodes

<!-- archie:ai-start -->

> Full CRUD v3 HTTP handler package for billing tax codes (list, get, create, update/upsert, delete). Uses Goverter-generated conversions (convert.gen.go) driven by annotated convert.go; includes custom AppType mapping (external_invoicing ↔ custom_invoicing) for backwards compatibility.

## Patterns

**Goverter variables pattern for conversions** — convert.go declares package-level var functions annotated with goverter directives. convert.gen.go is generated and must never be hand-edited. The //go:generate directive at the top of convert.go regenerates it. (`// goverter:variables
var FromAPICreateTaxCodeRequest func(namespace string, createTaxCodeRequest api.CreateTaxCodeRequest) (taxcode.CreateTaxCodeInput, error)`)
**Backwards-compatible AppType mapping** — FromAPIBillingAppType maps API value 'external_invoicing' to domain AppTypeCustomInvoicing. ToAPIBillingAppType maps domain AppTypeCustomInvoicing back to 'external_invoicing'. This pair must stay in sync. (`func FromAPIBillingAppType(source api.BillingAppType) app.AppType { if source == "external_invoicing" { return app.AppTypeCustomInvoicing } ... }`)
**Nil-to-empty-slice in ToAPIBillingTaxCodeAppMappings** — Return []api.BillingTaxCodeAppMapping{} (not nil) when source is nil to prevent JSON null in AppMappings field. (`func ToAPIBillingTaxCodeAppMappings(source taxcode.TaxCodeAppMappings) []api.BillingTaxCodeAppMapping { if source == nil { return []api.BillingTaxCodeAppMapping{} } ... }`)
**Goverter context for namespace injection** — Use goverter:context to pass namespace/namespacedID into generated converters. Implement NamespaceFromContext and ResolveNamespacedIDFromContext as context extractor functions. (`// goverter:context namespace
// goverter:map Namespace | NamespaceFromContext
var FromAPICreateTaxCodeRequest func(namespace string, ...) (...)`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `convert.go` | Source-of-truth for Goverter conversion declarations. Contains //go:generate directive, var declarations with goverter annotations, and hand-written helper functions (AppType mapping, nil-safe app mappings). | Editing annotations here requires running 'make generate' to regenerate convert.gen.go. Do NOT edit convert.gen.go directly. |
| `convert.gen.go` | Generated by Goverter from convert.go. DO NOT EDIT — has '// Code generated by github.com/jmattheis/goverter, DO NOT EDIT.' header. | If this file is out of sync with convert.go, builds will produce stale or incorrect conversions. |
| `handler.go` | Handler interface (5 methods) and handler struct with resolveNamespace and taxcode.Service only — minimal dependencies. | No secondary service needed; taxcode.Service handles all CRUD. |
| `update.go` | Uses UpsertTaxCodeRequest (not UpdateTaxCodeRequest) as the API body type — the endpoint is an upsert, not a partial update. | Operation name is 'upsert-tax-code' not 'update-tax-code' — keep consistent with TypeSpec. |

## Anti-Patterns

- Editing convert.gen.go directly — regenerate via 'make generate' instead
- Forgetting the AppType backwards-compat mapping — 'external_invoicing' must round-trip correctly
- Returning nil for AppMappings instead of empty slice — causes JSON null which breaks API contract

## Decisions

- **Goverter for tax code conversions instead of hand-written** — Tax code conversion is straightforward field mapping without discriminated unions; Goverter reduces boilerplate while the context mechanism cleanly injects namespace.
- **'external_invoicing' ↔ 'custom_invoicing' mapping in both directions** — The domain type was renamed from external_invoicing to custom_invoicing; the API preserves the old name for backwards compatibility with existing clients.
