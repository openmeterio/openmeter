# taxcodes

<!-- archie:ai-start -->

> Full CRUD v3 HTTP handler package for billing tax codes (list, get, create, update/upsert, delete) plus organization-default tax code management. Uses Goverter-generated conversions (convert.gen.go) driven by annotated convert.go; includes custom AppType mapping (external_invoicing <-> custom_invoicing) for backwards compatibility.

## Patterns

**Goverter variables pattern for conversions** — convert.go declares package-level var functions annotated with goverter directives. convert.gen.go is generated and must never be hand-edited. The //go:generate directive at the top of convert.go regenerates it via 'make generate'. (`// goverter:variables
var FromAPICreateTaxCodeRequest func(namespace string, createTaxCodeRequest api.CreateTaxCodeRequest) (taxcode.CreateTaxCodeInput, error)`)
**Backwards-compatible AppType mapping** — FromAPIBillingAppType maps API value 'external_invoicing' to domain AppTypeCustomInvoicing. ToAPIBillingAppType maps domain AppTypeCustomInvoicing back to 'external_invoicing'. This pair must stay in sync. (`func FromAPIBillingAppType(source api.BillingAppType) app.AppType { if source == "external_invoicing" { return app.AppTypeCustomInvoicing } ... }`)
**Nil-to-empty-slice in ToAPIBillingTaxCodeAppMappings** — Return []api.BillingTaxCodeAppMapping{} (not nil) when source is nil to prevent JSON null in AppMappings field. (`func ToAPIBillingTaxCodeAppMappings(source taxcode.TaxCodeAppMappings) []api.BillingTaxCodeAppMapping { if source == nil { return []api.BillingTaxCodeAppMapping{} } ... }`)
**Goverter context for namespace injection** — Use goverter:context to pass namespace/namespacedID into generated converters. Implement NamespaceFromContext and ResolveNamespacedIDFromContext as context extractor functions referenced by goverter:map directives. (`// goverter:context namespace
// goverter:map Namespace | NamespaceFromContext
var FromAPICreateTaxCodeRequest func(namespace string, ...) (...)`)
**Upsert semantics for update endpoint** — update.go uses UpsertTaxCodeRequest (not UpdateTaxCodeRequest) as the API body type — the endpoint is an upsert, not a partial update. Operation name is 'upsert-tax-code' not 'update-tax-code'. (`httptransport.WithOperationName("upsert-tax-code")`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `convert.go` | Source-of-truth for Goverter conversion declarations. Contains //go:generate directive, var declarations with goverter annotations, and hand-written helper functions (AppType mapping, nil-safe app mappings, reference ID extraction). | Editing annotations here requires running 'make generate' to regenerate convert.gen.go. Do NOT edit convert.gen.go directly. |
| `convert.gen.go` | Generated by Goverter from convert.go. Has '// Code generated by github.com/jmattheis/goverter, DO NOT EDIT.' header. | If this file is out of sync with convert.go annotations, builds produce stale/incorrect conversions. Regenerate with 'make generate'. |
| `handler.go` | Handler interface (7 methods including organization default tax code ops) and handler struct with resolveNamespace and taxcode.Service only. | No secondary service needed; taxcode.Service handles all CRUD including organization defaults. |
| `update.go` | Uses UpsertTaxCodeRequest as the API body type — the endpoint is an upsert. Passes models.NamespacedID to FromAPIUpsertTaxCodeRequest. | Operation name must match TypeSpec: 'upsert-tax-code'. Using api.UpdateTaxCodeRequest instead of api.UpsertTaxCodeRequest will break JSON deserialization. |

## Anti-Patterns

- Editing convert.gen.go directly — regenerate via 'make generate' after modifying goverter annotations in convert.go
- Forgetting the AppType backwards-compat mapping — 'external_invoicing' must round-trip correctly through both FromAPIBillingAppType and ToAPIBillingAppType
- Returning nil for AppMappings instead of empty slice — causes JSON null which breaks API contract for clients expecting an array
- Adding a new goverter:extend function in convert.go without also registering it in the global goverter:extend block at the top

## Decisions

- **Goverter for tax code conversions instead of hand-written** — Tax code conversion is straightforward field mapping without discriminated unions; Goverter reduces boilerplate while the context mechanism cleanly injects namespace.
- **'external_invoicing' <-> 'custom_invoicing' mapping in both directions** — The domain type was renamed from external_invoicing to custom_invoicing; the API preserves the old name for backwards compatibility with existing clients.

## Example: Adding a new Goverter conversion for a new request type following existing pattern

```
// In convert.go — add a new goverter:variables entry:
// goverter:context namespace
// goverter:map Namespace | NamespaceFromContext
// goverter:map Labels Metadata
// goverter:ignore Annotations
var FromAPINewTaxCodeRequest func(namespace string, req api.NewTaxCodeRequest) (taxcode.NewTaxCodeInput, error)

// Then run: make generate
// This regenerates convert.gen.go with the new converter function.
// In the operation file:
req, err := FromAPINewTaxCodeRequest(ns, body)
if err != nil { return Request{}, err }
```
