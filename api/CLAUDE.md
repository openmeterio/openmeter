# api

<!-- archie:ai-start -->

> Public API surface for OpenMeter: TypeSpec source (api/spec/) is the single source of truth that compiles to OpenAPI YAMLs, Go server stubs (api/api.gen.go, api/v3/api.gen.go), and Go/JS/Python SDKs (api/client/). All downstream artefacts are generated — never edit them directly. Primary constraint: any new endpoint must start in TypeSpec, flow through make gen-api, then make generate before a handler is written.

## Patterns

**Two-step regeneration mandatory** — TypeSpec changes require make gen-api (regenerates OpenAPI YAMLs + SDKs) then make generate (regenerates Go server stubs, Wire, Ent). Both steps must run and be committed together. (`make gen-api && make generate`)
**Generated-file guard** — Files with 'Code generated by X, DO NOT EDIT.' headers (api/api.gen.go, api/v3/api.gen.go, api/client/go/client.gen.go) must never be hand-edited. Edit api/spec/ then regenerate. (`// Code generated by github.com/oapi-codegen/oapi-codegen/v2 ... DO NOT EDIT.`)
**v1 vs v3 package split** — v1 API belongs in api/spec/packages/legacy/ producing api/api.gen.go; v3 AIP API belongs in api/spec/packages/aip/ producing api/v3/api.gen.go. Never mix content between versions. (`api/spec/packages/aip/src/openmeter.tsp for v3; api/spec/packages/legacy/src/main.tsp for v1`)
**Route and tag bindings at root namespace only** — @route and @tag decorators must only appear in root openmeter.tsp (v3) or main.tsp (v1), never inside domain sub-folder operation files. (`// In api/spec/packages/aip/src/openmeter.tsp: @route("/api/v1/meters") interface MeterRoutes {}`)
**Error responses via apierrors package** — All v3 error responses must use the apierrors package for RFC 7807 compliance. Direct http.Error or w.WriteHeader calls are forbidden in handlers. (`return nil, apierrors.NewValidationError("invalid input")`)
**api/types for manual gap-fill types only** — Only types that oapi-codegen cannot generate (oneOf/anyOf schemas) live in api/types via x-go-type references. No business logic, helpers, or methods go here. (`// api/types/doc.go — pure type definitions referenced from openapi spec via x-go-type`)
**nullable.Nullable[T] for optional JSON fields in v3** — v3 codegen uses nullable-type:true — use nullable.Nullable[string] not *string for optional JSON fields to preserve the null vs absent distinction. (`type UpdateInput struct { Name nullable.Nullable[string] }`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `api/api.gen.go` | Generated v1 server stubs, enums, request/response types from openapi.yaml via oapi-codegen | DO NOT EDIT — regenerate via make gen-api; hand edits are silently overwritten |
| `api/v3/api.gen.go` | Generated v3 server stubs from api/v3/openapi.yaml via oapi-codegen with chi-server and nullable types | DO NOT EDIT — regenerate via make gen-api; uses always-prefix-enum-values so enum consts are prefixed |
| `api/v3/codegen.yaml` | oapi-codegen config for v3: nullable-type=true, always-prefix-enum-values, custom chi-middleware.tmpl for deepObject filters | custom chi-middleware.tmpl patch must stay aligned with codegen.yaml settings when upgrading oapi-codegen |
| `api/spec/packages/legacy/src/main.tsp` | v1 TypeSpec entry point; all new legacy sub-domain .tsp files must be registered here AND in cloud/main.tsp | Missing registration means endpoint is not generated — check both main.tsp and cloud/main.tsp |
| `api/spec/packages/aip/src/openmeter.tsp` | v3 AIP TypeSpec entry point; @route and @tag bindings are declared here only, not in domain sub-folder files | @route and @tag must NOT appear inside domain sub-folder TypeSpec operation files |
| `api/client/go/client.gen.go` | Generated Go SDK from api/openapi.cloud.yaml (not openapi.yaml) — cloud variant exposes additional cloud-only endpoints | Must not import internal monorepo packages — stays externally importable; regenerated from cloud spec only |
| `api/openapi.yaml` | Generated v1 OpenAPI spec — downstream artifact of make gen-api from TypeSpec | Never hand-edit; always run make gen-api after TypeSpec changes |
| `api/spec/packages/aip/scripts/flatten-allof.mjs` | Post-processing script that flattens allOf schemas in the v3 OpenAPI output | Must run as part of make gen-api pipeline; bypassing it leaves unflattened allOf in api/v3/openapi.yaml |

## Anti-Patterns

- Hand-editing api/openapi.yaml, api/openapi.cloud.yaml, api/v3/openapi.yaml, or any *.gen.go file — always regenerate via make gen-api
- Adding @route or @tag inside domain sub-folder TypeSpec files — routing belongs in root namespace files (openmeter.tsp / main.tsp) only
- Adding business logic inside api/v3/ outside handlers/ sub-packages — api/v3 is infrastructure only; all domain calls belong in api/v3/handlers/*
- Adding new legacy sub-domain TypeSpec file without registering it in both main.tsp and cloud/main.tsp — endpoint silently absent
- Using *string instead of nullable.Nullable[string] for optional JSON fields in v3 — produces incorrect null vs absent distinction

## Decisions

- **TypeSpec as single source of truth compiling to OpenAPI + multi-language SDKs** — One spec feeds Go, JS, and Python SDKs plus two server versions — eliminates cross-language drift and forces breaking changes to be caught at TypeSpec compile time
- **Go SDK generated from api/openapi.cloud.yaml, not openapi.yaml** — Cloud variant exposes additional cloud-only endpoints not present in the self-hosted spec; keeps self-hosted and cloud SDKs correctly scoped
- **oapi-codegen with nullable-type:true and always-prefix-enum-values for v3** — Correct null vs absent distinction for optional JSON fields; prefixed enums prevent name collisions across the large v3 type namespace

## Example: Add a v3 API endpoint: author TypeSpec then implement handler

```
// 1. In api/spec/packages/aip/src/openmeter.tsp — declare the op
// 2. make gen-api && make generate
// 3. Implement in api/v3/handlers/foo/handler.go:
import (
    api "github.com/openmeterio/openmeter/api/v3"
    "github.com/openmeterio/openmeter/pkg/framework/transport/httptransport"
)
func (h *Handler) ListFoos(ctx context.Context, req api.ListFoosRequestObject) (api.ListFoosResponseObject, error) {
    items, err := h.svc.List(ctx, req.Params.Namespace)
    if err != nil { return nil, err }
    return api.ListFoos200JSONResponse{Items: toAPI(items)}, nil
}
```
