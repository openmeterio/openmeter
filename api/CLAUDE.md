# api

<!-- archie:ai-start -->

> Public API surface for OpenMeter: TypeSpec source (api/spec/) is the single source of truth that compiles to OpenAPI YAMLs, Go server stubs (api/api.gen.go, api/v3/api.gen.go), Go/JS/Python SDKs (api/client/). All downstream artefacts are generated — never edit them directly.

## Patterns

**Two-step regeneration** — TypeSpec changes require `make gen-api` (regenerates OpenAPI + SDKs) then `make generate` (regenerates Go server stubs, wire, ent). Both steps must run together. (`make gen-api && make generate`)
**Generated-file guard** — Files with 'Code generated by X, DO NOT EDIT.' headers (api/api.gen.go, api/v3/api.gen.go, api/client/go/client.gen.go) must never be hand-edited. (`// Code generated by github.com/oapi-codegen/oapi-codegen/v2 ... DO NOT EDIT.`)
**v1 vs v3 package split** — v1 API belongs in api/spec/packages/legacy/ and api/api.gen.go; v3 AIP API belongs in api/spec/packages/aip/ and api/v3/. Never mix content between versions. (`api/spec/packages/aip/src/openmeter.tsp for v3 resources`)
**api/types for manual gap-fill types** — Only types that oapi-codegen cannot generate (oneOf/anyOf schemas) live in api/types via x-go-type references. No business logic or helpers go here. (`api/types/doc.go — pure type definitions`)
**Error responses via apierrors** — All v3 error responses must go through the apierrors package for RFC 7807 compliance; direct http.Error or w.WriteHeader calls are forbidden. (`return nil, apierrors.NewValidationError(...)`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `api/api.gen.go` | Generated v1 server stubs, enums, request/response types from openapi.yaml via oapi-codegen | DO NOT EDIT — regenerate via make gen-api |
| `api/v3/api.gen.go` | Generated v3 server stubs from api/v3/openapi.yaml via oapi-codegen with chi-server + nullable types | DO NOT EDIT — regenerate via make gen-api |
| `api/v3/codegen.yaml` | oapi-codegen config for v3: nullable-type=true, always-prefix-enum-values, deepObject filter patch | custom chi-middleware.tmpl patch must stay aligned with codegen.yaml settings |
| `api/spec/packages/legacy/src/main.tsp` | v1 TypeSpec entry point; all new legacy sub-domain .tsp files must be registered here and in cloud/main.tsp | Missing registration = endpoint not generated |
| `api/spec/packages/aip/src/openmeter.tsp` | v3 AIP TypeSpec entry point; route/tag bindings are declared here only | @route and @tag must NOT be declared inside domain sub-folder operation files |
| `api/client/go/client.gen.go` | Generated Go SDK from api/openapi.cloud.yaml (not openapi.yaml) | Must not import internal monorepo packages — stays externally importable |
| `api/openapi.yaml` | Generated v1 OpenAPI spec | Never hand-edit; always regenerate via make gen-api |

## Anti-Patterns

- Hand-editing api/openapi.yaml, api/openapi.cloud.yaml, api/v3/openapi.yaml, or any *.gen.go file — always regenerate
- Adding @route or @tag inside domain sub-folder TypeSpec files — routing belongs in root namespace files only
- Adding business logic inside api/v3/ outside handlers/ sub-packages — api/v3 is infrastructure only
- Adding new API types to client wrapper files instead of api/spec/ TypeSpec
- Adding new legacy sub-domain TypeSpec file without registering in both main.tsp and cloud/main.tsp

## Decisions

- **TypeSpec as single source of truth compiling to OpenAPI + multi-language SDKs** — One spec feeds Go, JS, and Python SDKs plus two server versions — eliminates cross-language drift
- **Go SDK generated from api/openapi.cloud.yaml, not openapi.yaml** — Cloud variant exposes additional cloud-only endpoints not present in the self-hosted spec
- **oapi-codegen with nullable-type:true and always-prefix-enum-values for v3** — Correct null vs absent distinction for optional JSON fields; prefixed enums prevent name collisions

## Example: Add a v3 API endpoint: author TypeSpec then implement handler

```
// 1. In api/spec/packages/aip/src/openmeter.tsp — declare the op
// 2. make gen-api && make generate
// 3. Implement in api/v3/handlers/foo/handler.go:
func (h *Handler) ListFoos(ctx context.Context, req api.ListFoosRequestObject) (api.ListFoosResponseObject, error) {
    items, err := h.svc.List(ctx, req.Params.Namespace)
    if err != nil { return nil, err }
    return api.ListFoos200JSONResponse{Items: toAPI(items)}, nil
}
```
