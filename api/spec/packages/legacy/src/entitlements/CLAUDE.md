# entitlements

<!-- archie:ai-start -->

> TypeSpec schema-only package defining the v1 entitlements API surface — subject-scoped entitlement CRUD, grant management, and customer access checks. All V1 symbols are deprecated in favour of the v2/ sub-folder; new entitlement work belongs in v2/, not here.

## Patterns

**Discriminated union (envelope:none) for entitlement types** — The Entitlement read union and EntitlementCreateInputs create union both use @discriminated(#{ envelope: "none", discriminatorPropertyName: "type" }); a new entitlement type needs a new arm in BOTH unions plus a model with a matching type literal. (`union EntitlementCreateInputs { metered: ..., static: ..., boolean: ... }`)
**#deprecated + #suppress pair on every V1 symbol with a V2 successor** — Each V1 model/union/operation that has a V2 successor carries both #deprecated and #suppress; missing either causes compiler warnings that break make gen-api. (`#deprecated "Use EntitlementMeteredV2 instead"
#suppress "deprecated" "V1 Entitlements APIs will be removed on December 1st, 2025"`)
**OmitProperties<T,...> for read-model composition** — Response models spread OmitProperties over a base create-input type and add read-only/calculated fields, keeping create/read models in sync when the base changes. (`model EntitlementMetered { type: EntitlementType.metered; ...OmitProperties<EntitlementMeteredCreateInputs, "type" | "usagePeriod">; ...EntitlementMeteredCalculatedFields; }`)
**@operationId on every operation** — All operations carry @operationId for stable generated Go function names across compiler upgrades. (`@get @operationId("getCustomerAccess") @route("/access") getCustomerAccess(...): CustomerAccess | CommonErrors | NotFoundError;`)
**Separate interface group per URL prefix** — Each interface maps to exactly one URL prefix (@route + @tag) so new operations do not pollute another prefix's operation set. (`@route("/api/v1/customers/{customerIdOrKey}") @tag("Entitlements") interface CustomerEndpoints { ... }`)
**main.tsp imports all siblings + cross-package features** — Every new .tsp must be imported in main.tsp (which also cross-imports ../productcatalog/features.tsp) or it produces no output. (`import "./entitlements.tsp"; import "../productcatalog/features.tsp"; import "./grant.tsp"; import "./subjects.tsp"; import "./customer.tsp";`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `main.tsp` | Package entry importing all sibling .tsp files plus the cross-package productcatalog/features.tsp; no declarations live here. | Forgetting an import — the file compiles in isolation but is invisible to package output. |
| `entitlements.tsp` | Entitlement and EntitlementCreateInputs unions, EntitlementSharedFields, the three per-type create inputs, and admin endpoints at /api/v1/entitlements. | A new entitlement variant must update BOTH the read union AND the create union atomically or the discriminator is incomplete. |
| `subjects.tsp` | V1 subject-scoped entitlement CRUD and grant endpoints at /api/v1/subjects/{subjectIdOrKey}/entitlements; all deprecated. | Do not add new operations here — they belong in v2/; every operation carries #deprecated + #suppress. |
| `grant.tsp` | Grant, GrantCreateInput, ExpirationPeriod/Duration, and the global /api/v1/grants admin list/void endpoints; already imports @typespec/http. | New files referencing these models need import "@typespec/http" and using TypeSpec.Http or compilation fails. |
| `customer.tsp` | CustomerAccess and customer-scoped access-check endpoints (getCustomerAccess, getCustomerEntitlementValue) at /api/v1/customers/{customerIdOrKey}. | CustomerAccess.entitlements is Record<EntitlementValue> — splitting it into a typed map requires updating all SDK consumers. |

## Anti-Patterns

- Adding new V1 operations or models here instead of the v2/ sub-folder — all V1 entitlement APIs are deprecated
- Adding an entitlement variant to only one of Entitlement / EntitlementCreateInputs — both must change atomically
- Omitting the #deprecated + #suppress pair on new V1 symbols — causes warnings that break make gen-api
- Defining models without @friendlyName — autogenerated names can clash with V2 names in the same namespace
- Adding HTTP decorators in a .tsp file without import "@typespec/http" and using TypeSpec.Http — compilation failure

## Decisions

- **V1 symbols kept but fully deprecated in favour of v2/ equivalents** — A deprecation window is needed; #deprecated annotations let the compiler warn SDK consumers while the generator still emits the endpoints for backward-compatible clients.
- **OmitProperties composition instead of standalone read-model redefinitions** — Avoids maintaining parallel create/read hierarchies; the read model reflects create-input changes automatically unless explicitly omitted.
- **One shared EntitlementValue across all entitlement types with optional type-specific fields** — A uniform /value response shape (shared hasAccess + optional metered balance/usage/overage) reduces client branching.
