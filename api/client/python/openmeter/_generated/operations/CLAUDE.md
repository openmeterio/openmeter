# operations

<!-- archie:ai-start -->

> Auto-generated operation classes that implement every API endpoint as typed Python methods. Each Operations class groups related endpoints (CustomersOperations, InvoicesOperations, etc.) and uses corehttp pipeline to build, send, and deserialize HTTP requests.

## Patterns

**build_*_request helper functions** — Each operation is preceded by a free function build_<resource>_<verb>_request(...) that constructs the HttpRequest (URL, headers, params) using _SERIALIZER. The operation method calls this builder then sends via self._client.send_request. (`def build_customers_list_request(*, page=None, page_size=None, **kwargs) -> HttpRequest: ...`)
**ClsType callback pattern** — Every operation accepts an optional cls: ClsType kwarg — a callback(pipeline_response, deserialized, headers) that can intercept the result. Pass None for standard use. (`ClsType = Optional[Callable[[PipelineResponse[HttpRequest, HttpResponse], T, dict[str, Any]], Any]]`)
**map_error for HTTP error mapping** — Operations call map_error(status_code=response.status_code, response=response, error_map={404: ResourceNotFoundError, 409: ResourceExistsError, ...}) before deserializing success responses. (`map_error(status_code=response.status_code, response=response, error_map=error_map)`)
**_patch.py escape hatch** — _patch.py is intentionally empty but must be preserved. Method-level customizations to generated operations go here. (`# In _patch.py: override or extend a generated operation class method`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `_operations.py` | All generated operation classes (~30 classes). Each class is initialized with a PipelineClient and OpenMeterClientConfiguration. DO NOT EDIT — regenerated by make gen-api. | _SERIALIZER is a module-level singleton; do not create per-request Serializer instances. |
| `__init__.py` | Re-exports all Operations classes and applies _patch.py. The __all__ list is authoritative. | DO NOT EDIT — regenerated. _patch.__all__ must list any additional symbols added in _patch.py. |
| `_patch.py` | The only safe hand-edit point in this package. Customize or override generated operation methods here. | patch_sdk() must remain callable. |

## Anti-Patterns

- Editing _operations.py directly — regenerated by make gen-api
- Building HttpRequest objects manually outside the build_*_request pattern (breaks serialization consistency)
- Catching HttpResponseError without calling map_error first (loses typed exception mapping)

## Decisions

- **Request construction is split into build_*_request free functions separate from the send+deserialize logic in the Operations class methods.** — Enables unit-testing request construction without an HTTP client, and keeps the send path uniform across all operations.

## Example: Invoke a generated operation method

```
from openmeter._generated.operations import CustomersOperations
# ops is injected by the client; not constructed directly
result = ops.list(page=1, page_size=20)
for customer in result:
    print(customer.name)
```
