# _generated

<!-- archie:ai-start -->

> Root of the auto-generated Python SDK for OpenMeter. Exposes OpenMeterClient (sync) as the sole public entry point; all resource groups are attributes instantiated in __init__ with a shared PipelineClient, Serializer, and Deserializer. Never edit directly — regenerated by `make gen-api`.

## Patterns

**Single shared PipelineClient with fixed policy chain** — OpenMeterClient constructs one PipelineClient with a fixed policy list (headers, user-agent, proxy, ContentDecode, retry, auth, logging) and passes it to every Operations constructor. Never create a second PipelineClient inside an Operations class. (`self._client: PipelineClient = PipelineClient(endpoint=_endpoint, policies=_policies, **kwargs)`)
**Operations-as-attributes pattern** — Every resource group is instantiated as a named attribute in OpenMeterClient.__init__ receiving (self._client, self._config, self._serialize, self._deserialize). Access via client.customers.list_customers(), not by instantiating Operations directly. (`self.customers = CustomersOperations(self._client, self._config, self._serialize, self._deserialize)`)
**send_request URL formatting** — When calling client.send_request with a raw HttpRequest, the {endpoint} placeholder in the URL must be resolved via format_url before dispatch. The method deepcopies the request and calls self._client.format_url with the serialized endpoint. (`request_copy.url = self._client.format_url(request_copy.url, **path_format_arguments)`)
**_patch.py customization escape hatch** — _patch.py is the ONLY file preserved across regenerations. Add public symbols to __all__ there and implement patch_sdk() for post-generation monkey-patching. __init__.py extends __all__ from _patch_all automatically. (`from ._patch import __all__ as _patch_all; __all__.extend([p for p in _patch_all if p not in __all__])`)
**Context manager protocol on client** — OpenMeterClient implements __enter__/__exit__ delegating to self._client. Always use with-statement or call close() explicitly to release the underlying HTTP connection pool. (`def __enter__(self) -> Self: self._client.__enter__(); return self`)
**Union type aliases in _types.py** — _types.py defines tagged-union type aliases (e.g. App, Entitlement, RateCard) using TYPE_CHECKING guards to avoid circular imports. These are the canonical discriminated-union types used as parameter and return types across Operations methods. (`App = Union["_models.StripeApp", "_models.SandboxApp", "_models.CustomInvoicingApp"]`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `_client.py` | Sync entry point — constructs PipelineClient, policy chain, Serializer/Deserializer, and all Operations instances as attributes. | Generated file — any hand-edit is overwritten by make gen-api. Adding a new resource group requires regenerating, not editing here. |
| `_configuration.py` | Holds OpenMeterClientConfiguration: stores endpoint, polling_interval, and all corehttp policy instances. Injected into every Operations constructor. | authentication_policy defaults to None — callers must pass one via kwargs or requests will be unauthenticated. |
| `_patch.py` | Only file safe to edit; survives regeneration. patch_sdk() runs after __init__.py imports complete. | patch_sdk() must remain present and callable — removing it breaks __init__.py import. |
| `_types.py` | Defines tagged-union Python type aliases for discriminated types (App, Entitlement, RateCard, NotificationRule, etc.) used as parameter/return type annotations across Operations. | Generated file. Uses TYPE_CHECKING guard to avoid circular imports — do not import these at runtime outside of type annotations. |
| `_version.py` | Pins SDK version string used by sdk_moniker in OpenMeterClientConfiguration user-agent. | Value is 0.0.0 in source; real version is injected at release build time. Do not hardcode a real version here. |
| `__init__.py` | Public surface: re-exports OpenMeterClient and VERSION; merges _patch_all into __all__ and calls patch_sdk(). | Generated — do not add imports here. Use _patch.py for any custom exports. |

## Anti-Patterns

- Hand-editing _client.py, __init__.py, or _version.py — all are overwritten by make gen-api
- Instantiating Operations classes directly instead of accessing them through the client attributes (bypasses shared serializer/deserializer and config)
- Passing a raw URL string to send_request without the {endpoint} placeholder resolved via format_url — causes malformed URLs against the target endpoint
- Adding retry or authentication logic inside _patch.py instead of injecting a custom policy via the policies kwarg to OpenMeterClient
- Calling client.close() without first draining streaming responses — leaks HTTP connections in the underlying PipelineClient pool

## Decisions

- **All request-building logic lives in child operations/_operations.py build_*_request helpers; _client.py only wires the pipeline.** — Keeps the client thin and regenerable; operations contain all schema-specific serialization so regeneration scope is isolated to child packages.
- **OpenMeterClientConfiguration is a separate injectable class rather than raw kwargs.** — Allows callers to share or replace a config object (e.g. swap authentication_policy) without reconstructing the client.
- **_patch.py is the sole file preserved across regenerations as a customization boundary.** — Prevents hand-edits from being silently lost on the next gen-api run while still allowing SDK-level monkey-patching via patch_sdk().

## Example: Instantiate the sync client with a bearer-token auth policy and call a resource group method

```
from corehttp.runtime import policies
from openmeter._generated import OpenMeterClient

auth = policies.BearerTokenCredentialPolicy(credential, "scope")
client = OpenMeterClient(
    endpoint="https://api.openmeter.cloud",
    authentication_policy=auth,
)
with client:
    customers = client.customers.list_customers()
```
