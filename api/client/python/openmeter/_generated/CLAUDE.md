# _generated

<!-- archie:ai-start -->

> Root of the auto-generated Python SDK for OpenMeter. Exposes OpenMeterClient (sync) as the sole public entry point; every resource group is an attribute instantiated in __init__ sharing one PipelineClient, Serializer, and Deserializer. Generated by `make gen-api` — never hand-edit except _patch.py.

## Patterns

**Single shared PipelineClient with fixed policy chain** — OpenMeterClient.__init__ builds exactly one PipelineClient with a fixed policy list (headers, user-agent, proxy, ContentDecode, retry, auth, logging) and passes it to every Operations constructor. Never create a second PipelineClient inside an Operations class. (`self._client: PipelineClient = PipelineClient(endpoint=_endpoint, policies=_policies, **kwargs)`)
**Operations-as-attributes** — Each resource group is a named attribute set in __init__, receiving (self._client, self._config, self._serialize, self._deserialize). Access via client.customers.list_customers(), never by instantiating an Operations class directly. (`self.customers = CustomersOperations(self._client, self._config, self._serialize, self._deserialize)`)
**send_request {endpoint} URL formatting** — send_request deepcopies the request and resolves the {endpoint} placeholder via self._client.format_url before dispatch; a raw URL passed without this resolution produces a malformed request. (`request_copy.url = self._client.format_url(request_copy.url, **path_format_arguments)`)
**_patch.py customization escape hatch** — _patch.py is the ONLY file preserved across regenerations. Add public symbols to its __all__ and implement patch_sdk() for post-generation monkey-patching; __init__.py merges _patch_all into __all__ and calls patch_sdk() automatically. patch_sdk() must stay present and callable. (`from ._patch import __all__ as _patch_all; __all__.extend([p for p in _patch_all if p not in __all__])`)
**Context-manager protocol on client** — OpenMeterClient implements __enter__/__exit__ delegating to self._client; always use a with-statement or call close() to release the underlying HTTP connection pool, after draining any streaming responses. (`def __enter__(self) -> Self: self._client.__enter__(); return self`)
**Tagged-union type aliases in _types.py** — _types.py defines discriminated-union aliases (App, Entitlement, RateCard, NotificationRule, SubscriptionEditOperation, ...) under a TYPE_CHECKING guard to avoid circular imports; these are the canonical param/return annotations across Operations and must not be imported at runtime outside annotations. (`App = Union["_models.StripeApp", "_models.SandboxApp", "_models.CustomInvoicingApp"]`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `_client.py` | Sync entry point — constructs PipelineClient, policy chain, Serializer/Deserializer, and all Operations attributes; defines send_request, close, __enter__/__exit__. | Generated — any hand-edit is overwritten by make gen-api. Adding a resource group requires regeneration, not editing here. |
| `_configuration.py` | OpenMeterClientConfiguration: stores endpoint, polling_interval, and all corehttp policy instances; injected into every Operations constructor. | authentication_policy defaults to None — callers must pass one via kwargs or requests go out unauthenticated. |
| `_patch.py` | Only file safe to edit; survives regeneration. patch_sdk() runs after __init__.py imports complete. | patch_sdk() must remain present and callable — removing it breaks the __init__.py import. Put retry/auth via a custom policy kwarg, not business logic here. |
| `_types.py` | Defines tagged-union Python type aliases (App, Entitlement, RateCard, NotificationRule, etc.) used as param/return annotations across Operations. | Generated; uses TYPE_CHECKING guard — do not import these at runtime outside type annotations. |
| `_version.py` | Pins the SDK version string used by sdk_moniker in OpenMeterClientConfiguration's user-agent. | Value is 0.0.0 in source; the real version is injected at release build time. Do not hardcode a real version here. |
| `__init__.py` | Public surface: re-exports OpenMeterClient and VERSION; merges _patch_all into __all__ and calls patch_sdk(). | Generated — do not add imports here; use _patch.py for any custom exports. |
| `aio/` | Child: async (asyncio) variant — AsyncPipelineClient + async Operations; execution/deserialization only, all request-building delegated to the sync operations layer. | Same generated-file rules apply; await response.read() before raising on a streaming error to avoid leaking the connection. |

## Anti-Patterns

- Hand-editing _client.py, __init__.py, _version.py, or operations/_operations.py — all overwritten by make gen-api.
- Instantiating Operations classes directly instead of accessing them through client attributes — bypasses the shared serializer/deserializer and config.
- Passing a raw URL to send_request without resolving the {endpoint} placeholder via format_url — produces malformed URLs.
- Adding retry or authentication logic inside _patch.py instead of injecting a custom policy via the policies kwarg to OpenMeterClient.
- Calling client.close() without first draining streaming responses — leaks HTTP connections in the PipelineClient pool.

## Decisions

- **All request-building lives in child operations/_operations.py build_*_request helpers; _client.py only wires the pipeline.** — Keeps the client thin and regenerable; schema-specific serialization is isolated to child packages so regeneration scope stays contained.
- **OpenMeterClientConfiguration is a separate injectable class rather than raw kwargs.** — Lets callers share or replace a config object (e.g. swap authentication_policy) without reconstructing the client.
- **_patch.py is the sole file preserved across regenerations as a customization boundary.** — Prevents hand-edits from being silently lost on the next gen-api run while still allowing SDK-level monkey-patching via patch_sdk().

## Example: Instantiate the sync client with a bearer-token auth policy and call a resource group method

```
from corehttp.runtime import policies
from openmeter._generated import OpenMeterClient

auth = policies.BearerTokenCredentialPolicy(credential, "scope")
client = OpenMeterClient(
    endpoint="https://api.openmeter.cloud",
    authentication_policy=auth,
)
with client:
    customers = client.customers.list_customers()
```
