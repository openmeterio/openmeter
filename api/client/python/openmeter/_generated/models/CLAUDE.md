# models

<!-- archie:ai-start -->

> Auto-generated Python model classes (~200+ in _models.py) and string enums (_enums.py) for the entire OpenMeter API surface — the single source of truth for Python-typed API objects in the SDK. Only _patch.py is safe to modify.

## Patterns

**rest_field declaration with visibility and name mapping** — Each field is a class variable via rest_field(name='camelCaseName', visibility=[...], format='rfc3339') mapping snake_case to JSON camelCase. Required fields have no default; optional are Optional[T] = rest_field(...). visibility controls which CRUD operations include the field. (`created_at: datetime.datetime = rest_field(name='createdAt', visibility=['read'], format='rfc3339')`)
**Dual @overload __init__** — Every model has two @overload signatures: keyword args (IDE completion of required fields) and mapping: Mapping[str, Any] (raw-dict construction from parsed JSON). The real __init__ calls super().__init__(*args, **kwargs). (`@overload
def __init__(self, *, name: str, key: str) -> None: ...
@overload
def __init__(self, mapping: Mapping[str, Any]) -> None: ...`)
**CaseInsensitiveEnumMeta for all string enums** — Every enum in _enums.py inherits (str, Enum) with metaclass=CaseInsensitiveEnumMeta from corehttp.utils, enabling case-insensitive lookup so wire comparisons work regardless of server casing. (`class AppType(str, Enum, metaclass=CaseInsensitiveEnumMeta):
    STRIPE = 'stripe'`)
**_patch.py as the only safe customization point** — _patch.py is preserved across regenerations. Add helpers/monkey-patches/public symbols here; new symbols must be listed in _patch.__all__ to appear in models.__all__. patch_sdk() must remain callable. (`__all__ = ['my_helper']
def patch_sdk(): pass`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `_models.py` | All ~200+ generated dataclass-style model classes extending _Model. Authoritative definition of every API request/response shape. | DO NOT EDIT — regenerated by make gen-api. All customizations go in _patch.py. |
| `_enums.py` | All string enums used across models and operations; each uses CaseInsensitiveEnumMeta for wire-value matching. | DO NOT EDIT — regenerated. Never add a plain str Enum without CaseInsensitiveEnumMeta. |
| `__init__.py` | Re-exports all models/enums and applies _patch via _patch_sdk(); __all__ is the authoritative public API of this subpackage. | DO NOT EDIT — regenerated. New symbols added in _patch.py must appear in _patch.__all__ to propagate here. |
| `_patch.py` | The only hand-editable file; place customizations that must survive regeneration here. | patch_sdk() must remain callable even if empty; never remove it. |

## Anti-Patterns

- Editing _models.py or _enums.py directly — overwritten by make gen-api
- Subclassing generated models outside _patch.py — breaks regeneration idempotency
- Using a plain str Enum without CaseInsensitiveEnumMeta — breaks case-insensitive wire matching
- Adding public symbols to _patch.py without listing them in _patch.__all__ — they won't reach models.__all__
- Constructing models with positional arguments — use keyword args or a Mapping dict

## Decisions

- **All models are MutableMapping-backed (_Model) rather than plain dataclasses** — Enables dict-style access and JSON serialization without a separate to_dict() step, matching the corehttp pipeline and SdkJSONEncoder's iteration over items().
- **_patch.py is the sole customization point preserved across regenerations** — Generated code stays pristine for automated regeneration; _patch.py is a stable escape hatch that survives make gen-api without merge conflicts.

## Example: Construct a model using typed kwargs or a raw dict from parsed JSON

```
from openmeter._generated.models import Customer, AppType
customer = Customer(name='Acme', external_mapping_key='acme-001')
customer = Customer({'name': 'Acme', 'externalMappingKey': 'acme-001'})
app_type = AppType('STRIPE')  # == AppType.STRIPE
```
