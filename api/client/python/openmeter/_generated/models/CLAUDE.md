# models

<!-- archie:ai-start -->

> Auto-generated Python model classes and enums for the entire OpenMeter API surface. Every request/response body type (~200+ classes in _models.py) and every string enum (_enums.py) is defined here; this is the single source of truth for Python-typed API objects in the SDK. Do not edit _models.py or _enums.py — only _patch.py is safe to modify.

## Patterns

**rest_field attribute declaration with visibility and name mapping** — Each field is declared as a class variable using rest_field(name='camelCaseName', visibility=[...], format='rfc3339') to map Python snake_case to JSON camelCase. Required fields have no default; optional fields are Optional[T] with = rest_field(...). The visibility list controls which CRUD operations include the field. (`created_at: datetime.datetime = rest_field(name='createdAt', visibility=['read'], format='rfc3339')`)
**Dual @overload __init__ pattern** — Every model class has two @overload signatures: one with keyword args (for IDE completion of required fields), one with mapping: Mapping[str, Any] (for raw-dict construction from parsed JSON). The real __init__ always calls super().__init__(*args, **kwargs). (`@overload
def __init__(self, *, name: str, key: str) -> None: ...
@overload
def __init__(self, mapping: Mapping[str, Any]) -> None: ...
def __init__(self, *args, **kwargs) -> None:
    super().__init__(*args, **kwargs)`)
**CaseInsensitiveEnumMeta for all string enums** — All enums in _enums.py inherit from (str, Enum) with metaclass=CaseInsensitiveEnumMeta from corehttp.utils. This enables case-insensitive lookup by string value so wire comparisons work regardless of server casing. (`class AppType(str, Enum, metaclass=CaseInsensitiveEnumMeta):
    STRIPE = 'stripe'
    SANDBOX = 'sandbox'`)
**_patch.py as the only safe customization point** — _patch.py is preserved across regenerations. Add helper methods, monkey-patches, or additional public symbols here. patch_sdk() must remain callable. New symbols must also be listed in _patch.__all__ to appear in models.__all__. (`# _patch.py
__all__ = ['my_helper']
def my_helper(): ...
def patch_sdk(): pass  # must remain callable`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `_models.py` | Contains all ~200+ generated dataclass-style model classes extending _Model from _utils/model_base. The authoritative definition of every API request/response shape. | DO NOT EDIT — regenerated by make gen-api. All customizations must go in _patch.py. |
| `_enums.py` | All string enums used across models and operations. Every enum uses CaseInsensitiveEnumMeta for wire-value matching. | DO NOT EDIT — regenerated. Never add plain str Enum without CaseInsensitiveEnumMeta. |
| `__init__.py` | Re-exports all models and enums, applies _patch.py via _patch_sdk(). The __all__ list is the authoritative public API of this subpackage. | DO NOT EDIT — regenerated. New public symbols added in _patch.py must appear in _patch.__all__ to propagate here. |
| `_patch.py` | The only file safe to hand-edit. Place SDK customizations that must survive regeneration here. | patch_sdk() must remain callable even if empty. Never remove it. |

## Anti-Patterns

- Editing _models.py or _enums.py directly — overwritten by make gen-api
- Subclassing generated models outside of _patch.py — breaks regeneration idempotency
- Using plain str Enum without CaseInsensitiveEnumMeta — breaks case-insensitive wire value matching
- Adding new public symbols to _patch.py without listing them in _patch.__all__ — they won't appear in models.__all__
- Constructing models with positional arguments — use keyword args or a Mapping dict; positional args are not supported by the overload signatures

## Decisions

- **All models are MutableMapping-backed (_Model) rather than plain dataclasses** — Enables dict-style access and JSON serialization without a separate to_dict() step, matching the corehttp pipeline's expectation for request bodies and the SdkJSONEncoder's iteration over items().
- **_patch.py is preserved across regenerations as the sole customization point** — Generated code must remain pristine for automated regeneration; _patch.py provides a stable escape hatch that survives make gen-api without merge conflicts.

## Example: Construct a model using typed kwargs or a raw dict from parsed JSON

```
from openmeter._generated.models import Customer, AddonCreate, AppType

# Typed keyword construction (IDE completes required fields)
customer = Customer(name='Acme', external_mapping_key='acme-001')

# Raw dict construction from parsed JSON response
customer = Customer({'name': 'Acme', 'externalMappingKey': 'acme-001'})

# Enum usage — case-insensitive
app_type = AppType('STRIPE')  # == AppType.STRIPE
```
