# zod

<!-- archie:ai-start -->

> Generated Zod validation schemas for every OpenMeter Cloud API request/response shape. Produced by orval v8.7.0 from the OpenAPI spec — never edited manually. Provides runtime type-safety and coercion on top of the generated TypeScript types.

## Patterns

**Named constraint constants before schema** — Every regex, min, max, and default value is extracted as a named `export const` (e.g. `listAddonsQueryPageSizeMax = 1000`) before the schema object. Schemas reference these constants, never inline literals. (`export const listAddonsQueryPageSizeMax = 1000
// then inside schema:
pageSize: zod.coerce.number().max(listAddonsQueryPageSizeMax)`)
**zod.coerce for primitives** — All primitive fields use `zod.coerce.string()` / `zod.coerce.number()` / `zod.coerce.boolean()` to tolerate string-serialised query params from HTTP. (`page: zod.coerce.number().min(1).default(listAddonsQueryPageDefault)`)
**zod.union for discriminated request bodies** — Rate card and entitlement template variants use `zod.union([...])` with fully inlined object schemas — no shared sub-schema references within the file. (`rateCards: zod.array(zod.union([zod.object({...}), zod.object({...})]))`)
**Generated file — no manual edits** — Header `Generated by orval v8.7.0 🍺 Do not edit manually.` is authoritative. Regenerate via `make gen-api`. (`// Generated by orval v8.7.0 🍺
// Do not edit manually.`)

## Anti-Patterns

- Hand-editing index.ts — all changes must come from regenerating via `make gen-api` from the TypeSpec source
- Importing this file in Go server code — it is TypeScript only
- Adding custom validators outside index.ts in this folder — extend in the consuming application instead
- Relying on specific constant names in application code without re-exporting them, as names are derived from operation/parameter names which change with the spec

## Decisions

- **orval generates Zod schemas from OpenAPI rather than hand-authoring** — Keeps runtime validation in sync with the TypeSpec contract automatically; drift is impossible when make gen-api runs both steps.
- **Constraint values extracted as named exports** — Allows consuming code to import and reference the same boundary values (e.g. max page size) that the schema enforces, without duplicating magic numbers.
