# apiconverter

<!-- archie:ai-start -->

> Thin translation layer that converts v1 API filter/pagination types (api.Filter*) into internal pkg/filter and pkg/pagination/v2 types. All converter functions are Goverter-generated variables; manual code is limited to cursor decoding and the goverter directive file.

## Patterns

**Goverter variable declarations in filter.go** — Converter functions are declared as package-level var of function type with goverter directive comments. Never implement conversion logic by hand — add a new var entry with the appropriate goverter:ignore annotations and let `go generate` produce filter.gen.go. (`var ConvertString func(api.FilterString) filter.FilterString`)
**goverter:ignore for API-internal type mismatches** — When the internal filter type has fields not present in the v1 API type (e.g. Exists, Contains, Ncontains), add a `// goverter:ignore <Field>` directive above the var declaration. Remove the ignore when the v1 API grows the field. (`// goverter:ignore Exists Contains Ncontains
var ConvertString func(api.FilterString) filter.FilterString`)
**Nil-safe pointer wrappers** — For every converter there is a *Ptr variant (e.g. ConvertStringPtr) that handles nil input. Always expose both forms so callers never need nil-checks inline. (`ConvertStringPtr func(*api.FilterString) *filter.FilterString`)
**Cursor conversion delegates to pagination.DecodeCursor** — ConvertCursor and ConvertCursorPtr in cursor.go are hand-written thin wrappers over pagination.DecodeCursor. Keep cursor logic here, not in httpdriver packages. (`func ConvertCursor(s api.CursorPaginationCursor) (*pagination.Cursor, error) { return pagination.DecodeCursor(s) }`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `filter.go` | Source of truth for all converter declarations; contains only var declarations and goverter directives. Run `go generate ./` here to regenerate filter.gen.go. | Adding hand-written conversion logic here — all logic must live in the generated file. |
| `filter.gen.go` | Goverter-generated converter implementations. DO NOT EDIT — header says `Code generated by github.com/jmattheis/goverter, DO NOT EDIT.` | Manual edits; they will be overwritten on next `go generate`. |
| `cursor.go` | Hand-written cursor conversion using pagination.DecodeCursor. | Duplicating cursor decoding logic elsewhere in httpdriver packages. |

## Anti-Patterns

- Hand-writing conversion logic in filter.go instead of declaring a goverter var and regenerating
- Editing filter.gen.go directly — it is fully generated
- Adding domain business logic here; this package is a pure type-translation boundary
- Forgetting to add goverter:ignore for fields that exist on internal types but not on the v1 API type
- Adding v3 API types here — v3 has its own converter layer

## Decisions

- **Use Goverter variable-style converters instead of methods or standalone functions** — Variable-style converters allow recursive references (e.g. ConvertString calling itself for And/Or slices) and are initialized in a single init() block, keeping the generated code self-contained.
- **Keep cursor conversion hand-written rather than Goverter-generated** — pagination.DecodeCursor performs base64 decoding and cannot be mechanically derived by field mapping, so it must stay hand-written.

## Example: Adding a new filter type converter for a hypothetical FilterEnum API type

```
// In filter.go — add a new var with directives:
// goverter:ignoreMissing
var ConvertEnum    func(api.FilterEnum) filter.FilterEnum
var ConvertEnumPtr func(*api.FilterEnum) *filter.FilterEnum

// Then run: go generate ./
// This regenerates filter.gen.go with the new ConvertEnum and ConvertEnumPtr implementations.
```
