# stddetailedline

<!-- archie:ai-start -->

> Defines the Base struct for standard detailed billing line items (flat-fee and usage-based), the Ent schema mixin providing all DB columns, generic Creator/DBGetter interfaces for Ent mutations and mapping, and goderive-generated equality. This is the shared schema/model layer consumed by both Ent adapters and domain services — it bridges the Ent schema definition and the domain model in a single package.

## Patterns

**Composite Ent mixin via entutils.RecursiveMixin** — Mixin embeds entutils.RecursiveMixin[mixinBase] which composes entutils.AnnotationsMixin, entutils.ResourceMixin, and totals.Mixin. New fields must be added to mixinBase.Fields(), not directly to entity schemas. (`type Mixin struct { entutils.RecursiveMixin[mixinBase] }`)
**Generic Creator[T] interface for Ent create mutations** — Creator[T] composes externalid.LineExternalIDCreator[T], billingtotals.Setter[T], and all primitive Set* methods. The Create[T] function atomically populates all fields. New fields require updating both Creator[T] and the Create function in create.go. (`func Create[T Creator[T]](creator Creator[T], line Base) T`)
**DBGetter interface for DB-to-domain mapping** — FromDB[T DBGetter] reads all fields from a generic DB entity through the DBGetter interface. Adding a field to Base requires adding the getter to DBGetter and updating FromDB. All time.Time fields must be normalized to UTC with .In(time.UTC). (`func FromDB[T DBGetter](dbEntity T, taxConfig *productcatalog.TaxConfig) Base`)
**goderive equality generation — never hand-edit derived.gen.go** — Base.Equal delegates to deriveEqualBase generated by goderive. After adding fields to Base, run `go generate` in this package to regenerate derived.gen.go. The generator uses .Equal() for alpacadecimal.Decimal fields automatically. (`func (l Base) Equal(other Base) bool { return deriveEqualBase(&l, &other) }`)
**Deep clone for pointer and slice fields** — Base.Clone explicitly deep-copies TaxConfig pointer and CreditsApplied slice. Any new pointer or slice field added to Base must be deep-copied in Clone to prevent aliasing bugs. (`if l.TaxConfig != nil { taxConfig := *l.TaxConfig; l.TaxConfig = &taxConfig }`)
**Enum Values()/Validate() pattern with compile-time assertion** — Category implements models.Validator and uses slices.Contains against Category("").Values(). New enum types must follow the same Values()/Validate() pattern with var _ models.Validator = (*T)(nil). (`var _ models.Validator = (*Category)(nil)`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `model.go` | Canonical Base struct, Category enum, Comparable interface, Compare function, and BackfillTaxConfig helper. | Adding fields to Base requires updating: Clone, Equal (via goderive regen), DBGetter, Creator[T], FromDB, and mixin.go Fields(). All six locations must change together. |
| `mixin.go` | Ent schema mixin providing all DB fields for stddetailedline entities, SQL CHECK constraints (child_unique_reference_id NOT EMPTY), and indexes. | child_unique_reference_id has a NOT EMPTY SQL CHECK — never make it optional. credits_applied is jsonb-optional with pointer type in DB getter. |
| `create.go` | Generic Create[T] function for populating Ent create mutations from a Base value. | Must stay in sync with all Base fields; missing a field here silently omits it from DB writes. All time.Time values use .In(time.UTC). |
| `mapping.go` | DBGetter interface and FromDB generic mapping function from DB rows to Base. | time.UTC normalization must be applied to all time fields via .In(time.UTC). GetCreditsApplied returns a pointer — dereference with lo.FromPtr. |
| `derived.gen.go` | Goderive-generated equality helpers. DO NOT EDIT. | If Base gains fields with custom equality semantics (e.g. decimal), verify generated code uses .Equal() not == after regeneration. |
| `generate.go` | go:generate directive for goderive. | Run after any change to Base struct to keep derived.gen.go in sync. |

## Anti-Patterns

- Hand-editing derived.gen.go — always regenerate via go generate after Base changes.
- Adding DB/Ent logic to model.go or domain logic to mixin.go — keep layers strictly separate.
- Omitting time.UTC normalization in FromDB for time.Time fields — causes timezone-dependent test failures.
- Forgetting to update Creator[T] and DBGetter when adding a new field to Base — silently omits the field from writes/reads.
- Making child_unique_reference_id optional — it has a DB CHECK constraint and is required for line upsert identity.

## Decisions

- **Base struct co-located with the Ent mixin rather than in a separate domain package.** — Both the Ent schema definition and the domain model must stay in lock-step; co-locating them makes drift immediately visible and allows the Creator/DBGetter interfaces to share package-level types without cross-package indirection.
- **goderive for equality rather than hand-written Equal methods.** — Base contains alpacadecimal.Decimal fields that require .Equal() not ==; goderive generates correct field-by-field equality automatically and stays in sync with struct changes via go generate.

## Example: Add a new string field to Base and wire it through all layers

```
// 1. model.go: add field to Base struct
type Base struct {
    // ... existing fields
    NewField string `json:"newField"`
}
// Also update Clone() if the field is a pointer/slice.

// 2. mixin.go: add DB field to mixinBase.Fields()
field.String("new_field").NotEmpty()

// 3. create.go: add to Creator[T] interface and Create[T] function
type Creator[T any] interface {
    // ... existing methods
    SetNewField(string) T
}
// ...
```
