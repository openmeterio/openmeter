# stddetailedline

<!-- archie:ai-start -->

> Defines the Base struct for standard detailed billing line items, along with Ent mixin, DB mapping helpers, generic Creator interface for Ent mutations, and goderive-generated equality. This is the shared schema/model layer consumed by both Ent adapters and domain services for flat-fee and usage-based detailed lines.

## Patterns

**Composite Ent mixin via entutils.RecursiveMixin** — Mixin embeds entutils.RecursiveMixin[mixinBase] which composes entutils.AnnotationsMixin, entutils.ResourceMixin, and totals.Mixin. New fields must be added to mixinBase.Fields(), not directly to entity schemas. (`type Mixin struct { entutils.RecursiveMixin[mixinBase] }`)
**Generic Creator interface for Ent mutations** — Creator[T] composes externalid.LineExternalIDCreator[T], billingtotals.Setter[T], and all primitive Set* methods. The Create[T] function uses this to atomically populate all fields. New fields require updating both Creator[T] and the Create function. (`func Create[T Creator[T]](creator Creator[T], line Base) T`)
**DBGetter interface for DB-to-domain mapping** — FromDB[T DBGetter] reads all fields from a generic DB entity through the DBGetter interface. Adding a field to Base requires adding the corresponding getter to DBGetter and updating FromDB. (`func FromDB[T DBGetter](dbEntity T, taxConfig *productcatalog.TaxConfig) Base`)
**goderive equality generation** — Base.Equal delegates to deriveEqualBase generated by goderive. After adding fields to Base, run `go generate` in this package to regenerate derived.gen.go. Never hand-edit derived.gen.go. (`func (l Base) Equal(other Base) bool { return deriveEqualBase(&l, &other) }`)
**models.Validator on enums and structs** — Category implements models.Validator and uses slices.Contains against Category("").Values(). New enum types must follow the same Values()/Validate() pattern with compile-time assertion. (`var _ models.Validator = (*Category)(nil)`)
**Deep clone for pointer/slice fields** — Base.Clone explicitly deep-copies TaxConfig pointer and CreditsApplied slice. Any new pointer or slice field added to Base must be deep-copied in Clone. (`if l.TaxConfig != nil { taxConfig := *l.TaxConfig; l.TaxConfig = &taxConfig }`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `model.go` | Canonical Base struct, Category enum, Comparable interface, Compare function, and BackfillTaxConfig helper. | Adding fields to Base requires updating: Clone, Equal (via goderive regen), DBGetter, Creator[T], FromDB, and mixin.go Fields(). |
| `mixin.go` | Ent schema mixin providing all DB fields for stddetailedline entities. Also declares SQL CHECK constraints and indexes. | child_unique_reference_id has a NOT EMPTY SQL CHECK — never allow it to be optional. credits_applied is jsonb-optional. |
| `create.go` | Generic Create[T] function for populating Ent create mutations from a Base value. | Must stay in sync with all Base fields; missing a field here silently omits it from DB writes. |
| `mapping.go` | DBGetter interface and FromDB generic mapping function from DB rows to Base. | time.UTC normalization must be applied to all time fields via .In(time.UTC). |
| `derived.gen.go` | Goderive-generated equality helpers. DO NOT EDIT — regenerate with go generate. | If Base gains fields with custom equality semantics (e.g. decimal), the generated code uses .Equal() if available, otherwise == — verify correctness after regen. |
| `generate.go` | go:generate directive for goderive. | Run after any change to Base struct to keep derived.gen.go in sync. |

## Anti-Patterns

- Hand-editing derived.gen.go — always regenerate via go generate after Base changes.
- Adding DB/Ent logic directly to model.go or domain logic to mixin.go — keep the layers separate.
- Omitting time.UTC normalization in FromDB for time.Time fields — causes timezone-dependent test failures.
- Forgetting to update Creator[T] and DBGetter when adding a new field to Base — silently omits the field from writes/reads.
- Making child_unique_reference_id optional — it has a DB CHECK constraint and is required for line upsert identity.

## Decisions

- **Base struct lives in the same package as the Ent mixin rather than in a separate domain package.** — Both the Ent schema definition and the domain model must stay in lock-step; co-locating them makes drift immediately visible and allows the Creator/DBGetter interfaces to share the same package-level types.
- **goderive for equality rather than hand-written Equal methods.** — Base contains alpacadecimal.Decimal fields that require .Equal() not ==; goderive generates correct field-by-field equality automatically and stays in sync with struct changes via go generate.
