# models

<!-- archie:ai-start -->

> Structural container for shared billing domain model sub-packages: creditsapplied (credit slice type with currency-aware summation), externalid (invoice/line external-ID types and Ent mixin helpers), stddetailedline (detailed line base struct with Ent mixin and goderive equality), and totals (financial breakdown struct with Ent mixin and currency-precision rounding). No source files live here directly.

## Patterns

**Ent mixin co-located with domain model** — Each sub-package defines both the Go domain struct (model.go) and the corresponding Ent mixin (mixin.go) in the same package to keep schema and type in sync. (`// model.go: type Totals struct { ... }
// mixin.go: func (Mixin) Fields() []ent.Field { ... }`)
**Typed Setter/Getter generic interfaces over raw Ent mutations** — externalid and totals expose generic Creator[T]/Setter[T]/Getter interfaces so adapters never call individual Set* Ent methods directly. (`type Setter[T any] interface { SetTotals(T) *T }; func Set[T any](s Setter[T], t Totals) { ... }`)
**Currency-precision rounding before persistence** — Totals.RoundToPrecision(currency) must be called before any DB write to prevent numeric precision drift. (`t = t.RoundToPrecision(currencyx.Calculator{Currency: currency})`)
**models.Validator on enums and structs** — All enum types and structs in this package implement Validate() error conforming to models.Validator. (`func (s Status) Validate() error { switch s { case ...: return nil; default: return fmt.Errorf("invalid status: %s", s) } }`)
**goderive for equality on stddetailedline.Base** — derived.gen.go is generated by goderive from annotations in generate.go; never hand-edit the generated file. (`//go:generate go run github.com/awalterschulze/goderive . in generate.go`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `openmeter/billing/models/totals/model.go` | Totals struct with Add, Sum, RoundToPrecision, Validate. Validate enforces non-negative invariants. | Adding a field to Totals requires updating Add, RoundToPrecision, Validate, mixin Fields(), Setter[T], TotalsGetter, Set[T], and FromDB. |
| `openmeter/billing/models/externalid/mixin.go` | Ent mixin adding invoicing/payment external ID columns; generic Creator/Updater/Getter interfaces for type-safe Ent mutations. | Always use lo.EmptyableToPtr for nullable string columns — never store empty strings. |
| `openmeter/billing/models/stddetailedline/model.go` | Base struct for flat-fee and usage-based detailed lines. child_unique_reference_id is required (DB CHECK constraint). | child_unique_reference_id has a DB CHECK constraint — it is required for upsert identity; never make it optional. |
| `openmeter/billing/models/creditsapplied/model.go` | CreditsApplied slice with Validate (positivity) and currency-aware Sum. | Accumulating amounts without RoundToPrecision causes precision drift. |

## Anti-Patterns

- Adding DB/Ent logic to model.go or domain logic to mixin.go — keep layers strictly separated
- Hand-editing derived.gen.go — always regenerate via go generate
- Skipping RoundToPrecision before persisting Totals
- Storing empty strings in nullable external-ID columns instead of using lo.EmptyableToPtr
- Adding a new Totals field without updating all companion methods (Add, RoundToPrecision, Validate, mixin, Set, FromDB)

## Decisions

- **Mixin and domain model co-located in the same sub-package rather than in a shared mixin layer** — Keeps schema and domain type in sync; a single file change updates both the DB column and its Go representation.
- **goderive for equality rather than hand-written Equal methods on stddetailedline.Base** — Base has deeply nested pointer/slice fields; generated equality is complete and removes maintenance burden.
