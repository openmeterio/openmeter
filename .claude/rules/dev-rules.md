## Development Rules

### Anti Pattern

- Never place business logic in cmd/*/main.go — main.go only performs Wire initialization and run.Group startup; logic belongs in openmeter/ domain packages *(source: `cmd/server/main.go, cmd/billing-worker/main.go`)*
- Never spawn goroutines outside the oklog/run.Group in server/worker binaries — goroutines spawned outside it bypass graceful shutdown and leak resources *(source: `cmd/server/main.go (run.Group usage)`)*
- Never manually edit generated files: openmeter/ent/db/, *.gen.go, wire_gen.go, api/api.gen.go, api/v3/api.gen.go, api/client/go/client.gen.go, billing/derived.gen.go, convert.gen.go — all carry 'Code generated by X, DO NOT EDIT' headers *(source: `AGENTS.md Code Generation table, openmeter/ent/generate.go`)*
- Never hand-edit api/openapi.yaml, api/openapi.cloud.yaml, or api/v3/openapi.yaml, and never hand-edit SQL files in tools/migrate/migrations/ — both are generated (make gen-api / atlas migrate diff) and edits are overwritten or corrupt atlas.sum *(source: `Makefile (update-openapi, migrate-check), api/spec/CLAUDE.md`)*
- Never introduce context.Background() or context.TODO() to sidestep missing context propagation — propagate the caller's ctx through the full call path or remove the unused ctx parameter; the two legitimate exceptions are root ctx in main() and post-cancel graceful shutdown *(source: `AGENTS.md Coding Conventions, pkg/framework/entutils/transaction.go`)*
- Never use panic in non-test code — change the signature to return an error and propagate it; never use slog.Default() as a fallback dependency, require an injected *slog.Logger *(source: `AGENTS.md Coding Conventions`)*
- Never construct charges.Charge{}, charges.ChargeIntent{}, or billing.InvoiceLine{} as struct literals — use NewCharge[T]/NewChargeIntent[T]/NewStandardInvoiceLine/NewGatheringInvoiceLine, since the private discriminator stays zero-valued otherwise and accessors error *(source: `openmeter/billing/charges/service.go, openmeter/billing/service.go`)*
- Never mutate Invoice.Status or charge status fields directly — drive transitions through the stateless-backed InvoiceStateMachine or generic Machine[CHARGE,BASE,STATUS] so post-transition actions (DB save, event publish) fire *(source: `openmeter/billing/service/stdinvoicestate.go, openmeter/billing/charges/statemachine/machine.go`)*
- Never write hand-crafted SQL (db.Exec/db.Query) alongside Ent queries in the same adapter and never call Kafka admin APIs directly from domain code — topic provisioning goes through app/common's KafkaTopicProvisioner; this preserves the single-schema-source guarantee and Atlas diffing *(source: `openmeter/billing/adapter/adapter.go, app/common/, .archie/AGENTS.md`)*

### Boundary

- Never import app/common from a domain package under openmeter/ — the import direction is one-way (app/common imports domains); reversing creates import cycles and defeats the Wire compile-time graph *(source: `openmeter/CLAUDE.md, .archie/AGENTS.md, app/common/*.go`)*
- Never import app/common from openmeter/<domain>/testutils — build test dependencies from underlying package constructors (repos, adapters, services, lockr) directly, or unrelated wiring additions create test-only import cycles *(source: `AGENTS.md (Testing section), openmeter/*/testutils`)*

### Data Flow

- Always publish domain events through eventbus.Publisher (routes by EventName() prefix to IngestEventsTopic/SystemEventsTopic/BalanceWorkerEventsTopic); drive each EventName() prefix with a recognized EventVersionSubsystem constant or it silently misroutes to SystemEventsTopic — never publish to a Kafka topic by string literal *(source: `openmeter/watermill/eventbus/eventbus.go`)*
- Always use msg.Context() inside Watermill consumer handlers and silently drop unknown CloudEvents ce_types (return nil) in NoPublishingHandler; build consumer routers only via openmeter/watermill/router.NewDefaultRouter for the fixed middleware stack *(source: `openmeter/watermill/grouphandler/grouphandler.go, openmeter/watermill/router/router.go`)*
- Always maintain the sink worker three-phase flush order: ClickHouse BatchInsert -> Kafka offset commit -> Redis dedupe; fire FlushEventHandler.OnFlushSuccess in a goroutine, never synchronously inside flush() *(source: `openmeter/sink/sink.go`)*

### Pattern To Follow

- Always implement domain adapters with the TxCreator+TxUser triad (Tx via HijackTx+NewTxDriver, WithTx via NewTxClientFromRawConfig, Self) and wrap every method body in entutils.TransactingRepo / TransactingRepoWithNoValue so helpers rebind to the ctx-carried transaction; this includes helpers that accept a raw *entdb.Client in openmeter/billing/charges/.../adapter *(source: `AGENTS.md Coding Conventions, pkg/framework/entutils/transaction.go, openmeter/billing/charges/adapter/adapter.go`)*
- Always call lockr.Locker.LockForTX inside an active entutils.TransactingRepo-established Postgres transaction; never wrap it in context.WithTimeout (use pgdriver.WithLockTimeout); use charges.NewLockKeyForCharge / billing.Service.WithLock instead of hand-built lockr.Key strings *(source: `pkg/framework/lockr/locker.go, openmeter/billing/charges/lock.go`)*
- Always return models.Generic* typed sentinels (GenericNotFoundError, GenericValidationError, GenericConflictError) from service/adapter layers so commonhttp.GenericErrorEncoder maps them to RFC 7807 status codes; for Validate() collect issues into errs and return models.NewNillableGenericValidationError(errors.Join(errs...)) *(source: `pkg/models/errors.go, AGENTS.md Coding Conventions`)*
- Always build HTTP handlers via pkg/framework/transport/httptransport.NewHandler (decode -> operate -> encode); v1 handlers in openmeter/<domain>/httpdriver or httphandler, v3 handlers in api/v3/handlers/<resource>/ — never implement ServeHTTP directly *(source: `pkg/framework/transport/httptransport/handler.go, api/v3/handlers/*`)*
- Always prefer standard library and existing deps over local wrappers: slices.Clone for defensive copies, lo.ToPtr for pointer literals, lo.Must only in test setup; do not add ptr/loPtr/must/loMust wrappers when samber/lo covers the need *(source: `AGENTS.md Coding Conventions, go.mod (samber/lo v1.53.0)`)*
- Always use nullable.Nullable[T] (not *string) for optional JSON fields in v3 codegen, and use the apierrors package (not http.Error/w.WriteHeader) for v3 error responses *(source: `api/CLAUDE.md, api/v3/codegen.yaml (nullable-type:true)`)*
- Always run 'make gen-api' (TypeSpec compile + oapi-codegen + SDKs) AND then 'make generate' (go generate ./...) after editing TypeSpec in api/spec/; route/tag @decorators go only in the root openmeter.tsp/main.tsp, and adding @query/HTTP decorators requires importing @typespec/http with 'using TypeSpec.Http;' *(source: `AGENTS.md Code Generation, api/spec/CLAUDE.md, Makefile`)*
- Always run 'make generate' then 'atlas migrate --env local diff <name>' after editing openmeter/ent/schema/, committing ent code + .up.sql/.down.sql + atlas.sum together; new entities need IDMixin+NamespaceMixin+TimeMixin; ent.View schemas may need an explicit SQL migration since Atlas does not detect them *(source: `AGENTS.md Database Migrations, atlas.hcl, openmeter/ent/schema/*.go`)*
- Always build/test Go with -tags=dynamic (GO_BUILD_FLAGS) so confluent-kafka-go links librdkafka, and set POSTGRES_HOST=127.0.0.1 for DB tests or suites silently skip; use t.Context() over context.Background() in tests and pair clock.FreezeTime with defer clock.UnFreeze() *(source: `Makefile (GO_BUILD_FLAGS, test target), AGENTS.md Testing`)*
- Always drive usage-based billing lifecycle tests through charges.Service.Create/AdvanceCharges/ApplyPatches (not lower-level adapters) and use MockStreamingConnector with explicit StoredAt values to exercise stored-at cutoff logic in finalization *(source: `AGENTS.md Testing, openmeter/streaming/testutils`)*
- Always normalize LLM model IDs via llmcost.NormalizeModelID before storing or resolving prices, and truncate credit/entitlement effective times to time.Minute (Granularity) before passing to the credit engine *(source: `openmeter/llmcost/normalize.go, openmeter/credit`)*

### Wiring

- Always register cross-domain ServiceHooks and RequestValidators as side-effects inside app/common provider functions (e.g. customerService.RegisterHooks, RegisterRequestValidator, billing.Service.RegisterLineEngine) — never inside domain constructors; this avoids billing/customer/subscription/ledger import cycles *(source: `app/common/customer.go, app/common/billing.go, app/common/charges.go`)*
- Always guard credits.enabled=false at four independent wiring layers (ledger services in app/common/ledger.go, customer ledger hooks in app/common/customer.go, ChargesRegistry skip in app/common/billing.go, v3 credit handlers in api/v3/server) and return noop structs (ledgernoop.*) rather than nil; access charges via BillingRegistry.ChargesServiceOrNil() *(source: `app/common/ledger.go, app/common/customer.go, app/common/billing.go, AGENTS.md Configuration`)*
- Always add a matching app/common/openmeter_<binary>.go Wire provider set when adding a new cmd/* binary; cmd/<binary>/wire.go must list only app/common provider sets, not call domain constructors directly *(source: `app/common/openmeter_server.go, app/common/openmeter_billingworker.go, cmd/billing-worker/wire.go`)*
