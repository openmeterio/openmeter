## Development Rules

### Anti Pattern

- Never manually edit files inside openmeter/ent/db/ — they are generated by Ent from schemas in openmeter/ent/schema/ and must be regenerated with 'make generate'. *(source: `openmeter/ent/generate.go (//go:generate directive)`)*
- Never manually edit *.gen.go, wire_gen.go, api/api.gen.go, api/v3/api.gen.go, api/client/go/client.gen.go — all carry '// Code generated by X, DO NOT EDIT.' headers. *(source: `api/api.gen.go, api/v3/api.gen.go, cmd/*/wire_gen.go`)*
- Never hand-edit api/openapi.yaml, api/openapi.cloud.yaml, or api/v3/openapi.yaml — generated from TypeSpec source via 'make gen-api'. *(source: `Makefile (update-openapi target)`)*
- Never manually edit SQL migration files in tools/migrate/migrations/ — generated by Atlas and validated by CI via 'make migrate-check'; editing corrupts atlas.sum. *(source: `atlas.hcl, Makefile (migrate-check targets)`)*
- Never introduce context.Background() or context.TODO() in application code to work around missing context propagation — always propagate the caller's context through the full call path. *(source: `pkg/framework/entutils/transaction.go, .archie/AGENTS.md`)*
- Never pass a raw *entdb.Tx as a struct field or function parameter in adapters — always rebind to the context-carried transaction via entutils.TransactingRepo or TransactingRepoWithNoValue. *(source: `openmeter/billing/charges/adapter/adapter.go, pkg/framework/entutils/transaction.go`)*
- Never call a.db.Foo() directly in adapter method bodies in openmeter/billing/charges/**/adapter without wrapping in entutils.TransactingRepo — the raw client ignores the active Ent transaction in ctx. *(source: `openmeter/billing/charges/adapter/adapter.go`)*
- Never publish events to a Kafka topic by string literal — always use eventbus.Publisher from openmeter/watermill/eventbus which routes by EventName() prefix. *(source: `openmeter/watermill/eventbus/eventbus.go`)*
- Never construct a charges.Charge{} struct literal — always use NewCharge[T] and access via AsFlatFeeCharge/AsUsageBasedCharge/AsCreditPurchaseCharge; literal construction leaves the private discriminator empty. *(source: `openmeter/billing/charges/service.go`)*
- Never construct a billing.InvoiceLine{} struct literal — always use NewStandardInvoiceLine or NewGatheringInvoiceLine constructors; literal construction leaves the private discriminator t zero-valued. *(source: `openmeter/billing/service.go`)*
- Never add domain provider functions inside domain packages (openmeter/*) — all Wire provider sets must live in app/common/ to keep domain packages independent of the DI layer. *(source: `app/common/*.go, .archie/AGENTS.md`)*
- Never place business logic inside cmd/*/main.go — business logic belongs in domain packages under openmeter/; cmd/* only performs wiring via Wire and startup orchestration. *(source: `cmd/server/main.go, cmd/billing-worker/main.go`)*
- Never spawn goroutines outside the oklog/run.Group in worker and server binaries — goroutines spawned outside run.Group bypass graceful shutdown and can leak resources. *(source: `cmd/server/main.go (run.Group usage)`)*
- Never use HTTP calls or shared in-memory state between binaries for cross-binary communication — always use the three named Kafka topics via the Watermill eventbus.Publisher. *(source: `openmeter/watermill/eventbus/eventbus.go`)*
- Never return an error for unknown Watermill event types in a NoPublishingHandler — unknown ce_type must be silently dropped to support rolling deploys. *(source: `openmeter/watermill/grouphandler/`)*
- Never write hand-crafted SQL (db.Exec, db.Query) alongside Ent queries in the same adapter — this breaks the single-schema-source guarantee and undermines Atlas diffing. *(source: `openmeter/billing/adapter/adapter.go`)*
- Never add a new endpoint only in the Go handler package without first adding it to the TypeSpec source in api/spec/ — this bypasses the SDK contract pipeline. *(source: `Makefile (gen-api target), api/spec/packages/aip/`)*

### Boundary

- Never have domain packages under openmeter/ import app/common — the import direction is one-way outward (app/common imports domain packages); reversing creates import cycles. *(source: `openmeter/CLAUDE.md`)*
- Never add a new cmd/* worker binary without a matching app/common/openmeter_<binary>.go Wire provider set file. *(source: `app/common/openmeter_billingworker.go, app/common/openmeter_server.go`)*
- Never call Kafka admin APIs directly from domain code — topic provisioning must go through app/common's KafkaTopicProvisioner. *(source: `app/common/, .archie/AGENTS.md`)*

### Data Flow

- Always drive EventName() prefixes with a recognised EventVersionSubsystem constant so eventbus.GeneratePublishTopic routes to the correct Kafka topic — an unrecognised prefix silently misroutes to SystemEventsTopic. *(source: `openmeter/watermill/eventbus/eventbus.go`)*
- Always maintain strict three-phase flush ordering in the sink worker: ClickHouse BatchInsert first, then Kafka offset commit, then Redis deduplication update — reordering breaks exactly-once semantics. *(source: `openmeter/sink/sink.go`)*

### Pattern To Follow

- Always run 'make gen-api' (TypeSpec compile + oapi-codegen) then 'make generate' (go generate ./...) after editing TypeSpec files in api/spec/ — both steps are mandatory to keep Go stubs and SDKs in sync. *(source: `Makefile (gen-api and generate-all targets)`)*
- Always run 'make generate' (go generate ./...) after editing Ent schemas in openmeter/ent/schema/, Wire providers in app/common/, or Goverter converter interfaces before building or testing. *(source: `Makefile (generate target)`)*
- Always run 'atlas migrate --env local diff <name>' after editing Ent schema to create timestamped .up.sql/.down.sql files and update atlas.sum — all three artifacts must be committed together. *(source: `atlas.hcl, Makefile (migrate-diff)`)*
- Always build and test with '-tags=dynamic' (GO_BUILD_FLAGS=-tags=dynamic) to link confluent-kafka-go against the local librdkafka — omitting this tag causes link failures. *(source: `Makefile (GO_BUILD_FLAGS = -tags=dynamic)`)*
- Always set POSTGRES_HOST=127.0.0.1 when running tests that touch the database — without it many test suites silently skip even when PostgreSQL is running. *(source: `Makefile (test target: POSTGRES_HOST=127.0.0.1 go test)`)*
- Always wrap Ent access in openmeter/<domain>/adapter with entutils.TransactingRepo(ctx, a, func(...) ...) or TransactingRepoWithNoValue on every method body so helpers rebind to the transaction already carried in ctx. *(source: `pkg/framework/entutils/transaction.go`)*
- Always use 't.Context()' instead of 'context.Background()' in tests when a *testing.T or testing.TB is available to keep cancellation tied to the test harness. *(source: `.archie/AGENTS.md (Testing section)`)*
- Always use msg.Context() inside Watermill consumer handlers — never substitute context.Background() as it severs OTel spans and drops the Ent transaction. *(source: `openmeter/watermill/grouphandler/`)*
- Always register namespace handlers (LedgerNamespaceHandler, KafkaIngestNamespaceHandler) before initNamespace() in cmd/server/main.go so they receive CreateNamespace for the default namespace. *(source: `cmd/server/main.go`)*
- Always use 'nix develop --impure .#ci -c <command>' when Go, golangci-lint, Atlas, or other toolchain binaries are missing from the ambient shell — the Nix CI shell pins all toolchain versions. *(source: `.github/workflows/ci.yaml, flake.nix`)*
- Always use models.Generic* typed sentinels (GenericNotFoundError, GenericValidationError, GenericConflictError, etc.) for domain errors so GenericErrorEncoder maps them to correct RFC 7807 HTTP status codes. *(source: `pkg/models/errors.go, pkg/framework/commonhttp/`)*
- Always use BillingRegistry.ChargesServiceOrNil() — never access BillingRegistry.Charges directly, as it is nil when credits.enabled=false. *(source: `app/common/billing.go`)*
- Always call lockr.Locker.LockForTX inside an active entutils.TransactingRepo-established transaction — calling outside a transaction does not correctly scope the pg_advisory_xact_lock. *(source: `pkg/framework/lockr/locker.go`)*

### Wiring

- Always register cross-domain hooks and validators inside app/common provider functions as side-effects, not inside domain package constructors — this avoids circular imports between billing, customer, subscription, and ledger. *(source: `app/common/customer.go, app/common/billing.go`)*
- Always guard credits.enabled=false at four independent wiring layers: (1) ledger services in app/common/ledger.go, (2) customer ledger hooks in app/common/customer.go, (3) ChargesRegistry skipped in app/common/billing.go, (4) credit handlers in api/v3/server — a single guard is insufficient. *(source: `app/common/ledger.go, app/common/customer.go, app/common/billing.go, api/v3/server/server.go`)*
- Always register new charge type line engines with billing.Service.RegisterLineEngine() in app/common/charges.go — never call RegisterLineEngine from domain packages or cmd/* binaries. *(source: `app/common/charges.go`)*
- Always use typed registry structs (BillingRegistry, AppRegistry, SubscriptionServiceWithWorkflow) to group cohesive services rather than adding individual service fields to router.Config. *(source: `app/common/billing.go, openmeter/server/router/router.go`)*
