## Development Rules

### Anti Pattern

- Never edit files with a '// Code generated by X, DO NOT EDIT.' header — these include openmeter/ent/db/, **/wire_gen.go, **/convert.gen.go, billing/derived.gen.go, api/api.gen.go, api/v3/api.gen.go, api/client/go/client.gen.go *(source: `/Users/hamutarto/DEV/gbr/openmeter/AGENTS.md (Code Generation section)`)*
- Never introduce 'context.Background()' or 'context.TODO()' to work around missing context propagation in application code — always propagate the caller's context through the full call path *(source: `/Users/hamutarto/DEV/gbr/openmeter/AGENTS.md (Coding Conventions section)`)*
- Never manually edit files in tools/migrate/migrations/ — they are generated by Atlas and validated by CI via 'make migrate-check' *(source: `/Users/hamutarto/DEV/gbr/openmeter/Makefile (migrate-check target) and atlas.hcl`)*
- Never hand-edit api/openapi.yaml, api/openapi.cloud.yaml, api/v3/openapi.yaml — they are generated from TypeSpec source in api/spec/ via 'make gen-api' *(source: `/Users/hamutarto/DEV/gbr/openmeter/AGENTS.md (Code Generation table)`)*
- Never edit openmeter/ent/db/ directly — it is fully generated by 'go generate ./openmeter/ent/...' from schemas in openmeter/ent/schema/ *(source: `/Users/hamutarto/DEV/gbr/openmeter/AGENTS.md (Code Generation section)`)*

### Boundary

- Never place business logic inside cmd/*/main.go — business logic belongs in domain packages under openmeter/; cmd/* only performs wiring via Wire and startup *(source: `/Users/hamutarto/DEV/gbr/openmeter/AGENTS.md (Architecture section) and CLAUDE.md (Key trade-offs)`)*
- Never add domain provider functions inside domain packages (openmeter/*) — all Wire provider sets must live in app/common/ to keep domain packages independent of the DI layer *(source: `/Users/hamutarto/DEV/gbr/openmeter/AGENTS.md (Key Decisions: Google Wire DI)`)*
- Never add a new worker binary without a matching app/common/openmeter_*worker.go Wire provider set file *(source: `/Users/hamutarto/DEV/gbr/openmeter/AGENTS.md (Key trade-offs: Multi-binary orchestration)`)*

### Data Flow

- Always drive charge lifecycle tests through charges.Service.Create, AdvanceCharges, and ApplyPatches rather than calling lower-level charge adapters directly *(source: `/Users/hamutarto/DEV/gbr/openmeter/AGENTS.md (Testing section)`)*

### Pattern To Follow

- Always run 'make generate' (go generate ./...) after editing Ent schemas, Wire providers, or Goverter converter interfaces before building or testing *(source: `/Users/hamutarto/DEV/gbr/openmeter/Makefile (generate target) and AGENTS.md (Code Generation section)`)*
- Always run 'make gen-api' after editing TypeSpec files in api/spec/ to regenerate OpenAPI spec and SDKs before running 'make generate' *(source: `/Users/hamutarto/DEV/gbr/openmeter/Makefile (gen-api target calls update-openapi + generate-javascript-sdk)`)*
- Always build with '-tags=dynamic' (GO_BUILD_FLAGS=-tags=dynamic) to link confluent-kafka-go against the local librdkafka — omitting this tag causes build failures *(source: `/Users/hamutarto/DEV/gbr/openmeter/Makefile (GO_BUILD_FLAGS = -tags=dynamic)`)*
- Always set POSTGRES_HOST=127.0.0.1 when running tests that touch the database — without it many test suites will be skipped even if PostgreSQL is running *(source: `/Users/hamutarto/DEV/gbr/openmeter/Makefile (test target: POSTGRES_HOST=127.0.0.1 go test ...)`)*
- Always start PostgreSQL before running tests: 'docker compose up -d postgres'; the make test target checks PostgreSQL availability and exits with an error if it is not running *(source: `/Users/hamutarto/DEV/gbr/openmeter/Makefile (test target PGPASSWORD check)`)*
- Always wrap Ent access in openmeter/billing/charges/.../adapter with entutils.TransactingRepo(...) / TransactingRepoWithNoValue(...) even in shared helpers so helpers rebind to the transaction already carried in ctx *(source: `/Users/hamutarto/DEV/gbr/openmeter/AGENTS.md (Coding Conventions section)`)*
- Always use 't.Context()' instead of 'context.Background()' in tests when a *testing.T or testing.TB is available to keep cancellation tied to the test harness *(source: `/Users/hamutarto/DEV/gbr/openmeter/AGENTS.md (Testing section)`)*
- Always run 'make generate' then 'atlas migrate --env local diff <name>' after editing Ent schema to create timestamped .up.sql/.down.sql files and update atlas.sum — all three files must be committed *(source: `/Users/hamutarto/DEV/gbr/openmeter/AGENTS.md (Database Migrations section) and atlas.hcl`)*
- Always keep domain test helpers under openmeter/.../testutils independent from app/common — build test dependencies from underlying package constructors to avoid import cycles *(source: `/Users/hamutarto/DEV/gbr/openmeter/AGENTS.md (Testing section)`)*
- When adding @query or other HTTP decorators to a TypeSpec file that does not already use them, always import @typespec/http and add 'using TypeSpec.Http;' in that file — otherwise compilation fails *(source: `/Users/hamutarto/DEV/gbr/openmeter/AGENTS.md (Code Generation section)`)*
- Always register namespace handlers before initNamespace(...) in cmd/server/main.go if they must provision the default namespace during startup *(source: `/Users/hamutarto/DEV/gbr/openmeter/AGENTS.md (Architecture section)`)*
- Always use 'nix develop --impure .#ci -c <command>' when Go, gofmt, or other toolchain binaries are missing from the ambient shell — the Nix CI shell pins all toolchain versions *(source: `/Users/hamutarto/DEV/gbr/openmeter/AGENTS.md (Testing section) and flake.nix`)*

### Wiring

- Always guard credits.enabled=false at multiple independent layers: ledger-backed credit handlers in api/v3/server, customer ledger hooks, namespace/default-account provisioning, and app/common noop fallback — a single guard is insufficient *(source: `/Users/hamutarto/DEV/gbr/openmeter/AGENTS.md (Configuration section)`)*
