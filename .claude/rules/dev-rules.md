## Development Rules

### Anti Pattern

- Never put business logic in cmd/*/main.go — main.go only does Wire init + oklog/run.Group lifecycle; logic belongs in openmeter/ domain packages *(source: `cmd/server/main.go, Dockerfile build targets, layer-002`)*
- Never spawn goroutines outside the oklog/run.Group in server/worker binaries — bypasses graceful shutdown and leaks resources *(source: `cmd/*/main.go (run.Group), dep-004`)*
- Never manually edit generated files: openmeter/ent/db/, *.gen.go, wire_gen.go, api/api.gen.go, api/v3/api.gen.go, api/client/go/client.gen.go, convert.gen.go (all carry 'Code generated, DO NOT EDIT' headers) *(source: `openmeter/ent/db, api/v3/api.gen.go header, gen-001/gen-002`)*
- Never hand-edit api/openapi.yaml, api/openapi.cloud.yaml, api/v3/openapi.yaml or SQL files in tools/migrate/migrations/ — regenerate via make gen-api / atlas migrate diff (edits are overwritten / corrupt atlas.sum) *(source: `api/spec/Makefile, atlas.hcl, gen-006/api-002`)*
- Never introduce context.Background()/context.TODO() in openmeter/ or cmd/* to sidestep ctx propagation — propagate the caller's ctx; the only exceptions are root ctx in main() and post-cancel graceful shutdown *(source: `pkg/framework/entutils/transaction.go, tx-002/ctx-003`)*
- Never construct charges.Charge{}, charges.ChargeIntent{}, or billing.InvoiceLine{} as struct literals — use NewCharge[T]/NewChargeIntent[T]/NewStandardInvoiceLine/NewGatheringInvoiceLine (private discriminator stays zero otherwise) *(source: `openmeter/billing/charges/charge.go, billing-003/005/008`)*
- Never mutate Invoice.Status or charge status fields directly — drive transitions through the stateless InvoiceStateMachine or generic Machine[CHARGE,BASE,STATUS] (qmuntal/stateless) so post-transition save+publish fire *(source: `openmeter/billing/service/stdinvoicestate.go, billing-002`)*
- Never write hand-crafted SQL (db.Exec/db.Query) alongside Ent queries in the same adapter; never call Kafka admin APIs directly from domain code — topic provisioning goes through app/common KafkaTopicProvisioner *(source: `openmeter/billing/adapter/adapter.go, ent-001/kafka-001`)*

### Boundary

- Never import app/common from a domain package under openmeter/ — Wire wiring flows one-way outward (app/common imports domains); reversing creates import cycles *(source: `app/common/*.go imports openmeter/* domains; dep-001`)*
- Never import app/common from openmeter/<domain>/testutils — build test deps from underlying constructors directly to avoid test-only import cycles *(source: `openmeter/*/testutils; test-003`)*
- Always define the domain Service and Adapter interface at the package root (service.go / adapter.go); concrete service in <domain>/service/, Ent adapter in <domain>/adapter/, v1 HTTP in httpdriver/, v3 HTTP in api/v3/handlers/<resource>/ *(source: `layer-001, layer-004, openmeter/<domain> layout`)*

### Data Flow

- Always publish domain events through eventbus.Publisher (routes by EventName() prefix to IngestEventsTopic/SystemEventsTopic/BalanceWorkerEventsTopic); drive each EventName() with a recognized EventVersionSubsystem prefix or it silently misroutes to SystemEventsTopic — never publish by string literal *(source: `openmeter/watermill/eventbus/eventbus.go, wm-001/pf-007`)*
- Always use msg.Context() inside Watermill consumer handlers, silently drop (return nil) unknown CloudEvents ce_types in NoPublishingHandler, and build consumer routers only via openmeter/watermill/router.NewDefaultRouter for the fixed middleware stack *(source: `openmeter/watermill/grouphandler/grouphandler.go, router/router.go, wm-002/wm-003/watermill-001`)*
- Always maintain the sink worker three-phase flush order: ClickHouse BatchInsert -> Kafka offset commit -> Redis dedupe; fire FlushEventHandler.OnFlushSuccess in a goroutine, never synchronously inside flush() *(source: `openmeter/sink/sink.go, sink-001/sink-002/dedupe-001`)*
- Always flow CloudEvent ingestion through ingest.Collector (handles dedupe + Kafka forwarding) and use meter.ParseEvent for value/group-by extraction — never write ingest events directly to ClickHouse or re-implement JSONPath inline *(source: `openmeter/ingest, openmeter/meter, ingest-001/meter-001`)*
- Always read every Ent-backed entity through its domain Service backed by the adapter wrapped in entutils.TransactingRepo; never query openmeter/ent/db directly from a service or any caller outside the adapter *(source: `openmeter/*/service/*.go, ent-004`)*

### Pattern To Follow

- Always implement adapters with the TxCreator+TxUser triad (Tx via HijackTx+NewTxDriver, WithTx via NewTxClientFromRawConfig, Self) and wrap every method body in entutils.TransactingRepo / TransactingRepoWithNoValue (including helpers that accept a raw *entdb.Client) *(source: `pkg/framework/entutils/transaction.go, openmeter/billing/charges/adapter, layer-007/ctx-001/ctx-002`)*
- Always call lockr.Locker.LockForTX inside an active entutils.TransactingRepo Postgres transaction; never wrap in context.WithTimeout (use pgdriver.WithLockTimeout); use charges.NewLockKeyForCharge / billing.Service.WithLock instead of hand-built lockr.Key strings *(source: `pkg/framework/lockr/locker.go, openmeter/billing/charges/lock.go, lock-002/003/004`)*
- Always return models.Generic* typed sentinels (GenericNotFoundError->404, GenericValidationError->400, GenericConflictError->409, etc.) from service/adapter layers so commonhttp.GenericErrorEncoder maps to RFC 7807; collect Validate() issues into errors.Join via models.NewNillableGenericValidationError *(source: `pkg/models/errors.go, pkg/framework/commonhttp/encoder.go, http-002/error-001`)*
- Always build HTTP handlers via pkg/framework/transport/httptransport.NewHandler (decode->operate->encode) — never implement ServeHTTP directly; v1 handlers in openmeter/<domain>/httpdriver, v3 in api/v3/handlers/<resource>/ *(source: `pkg/framework/transport/httptransport/handler.go, http-001/layer-005`)*
- Always use nullable.Nullable[T] (not *T) for optional v3 JSON fields (codegen.yaml nullable-type:true) and reference fully-prefixed enum constants (always-prefix-enum-values:true) with their Valid() method; use the apierrors package for v3 errors *(source: `api/v3/codegen.yaml, api/v3/api.gen.go`)*
- Always prefer stdlib and existing deps over local wrappers: slices.Clone, lo.ToPtr, samber/lo helpers, lo.Must only in test setup — do not add ptr/must wrappers (go.mod samber/lo v1.53.0) *(source: `go.mod, .archie skill samber-lo`)*
- Always truncate credit/entitlement effective times to time.Minute (Granularity) before passing to the credit engine, and normalize LLM model IDs via llmcost.NormalizeModelID before store/resolve; Grant.amount is Immutable *(source: `openmeter/credit, openmeter/llmcost/normalize.go, credit-001/llmcost-001`)*
- Always prefer pkg/framework/tracex.Start/Wrap over raw tracer.Start (centralises error recording + panic recovery); inject *slog.Logger and trace.Tracer rather than using slog.Default(); never panic in non-test code *(source: `pkg/framework/tracex, otel-001/otel-002`)*

### Wiring

- Always register cross-domain ServiceHooks and customer RequestValidators as side-effects inside app/common provider functions (customerService.RegisterHooks, RegisterRequestValidator, billing.Service.RegisterLineEngine) — never inside domain constructors (avoids billing/customer/subscription/ledger import cycles) *(source: `app/common/customer.go, app/common/billing.go, app/common/charges.go, dep-003/dep-006`)*
- Always guard credits.enabled=false at four wiring layers (ledger services in app/common/ledger.go, customer ledger hooks in app/common/customer.go, ChargesRegistry skip in app/common/billing.go, v3 credit handlers in api/v3/server) returning noop structs (ledgernoop.*) not nil; access charges via BillingRegistry.ChargesServiceOrNil() *(source: `app/common/ledger.go, app/common/billing.go, credits-001..004/di-001`)*
- Always add a matching app/common/openmeter_<binary>.go Wire provider set when adding a new cmd/* binary; cmd/<binary>/wire.go must list only app/common provider sets, not call domain constructors directly *(source: `app/common/openmeter_server.go, cmd/billing-worker/wire.go, dep-002/wire-001`)*
- Always register all namespace.Handler implementations (Ledger, KafkaIngest) in cmd/server before initNamespace; worker binaries must fail-fast assert the default namespace exists before namespace-scoped provisioning (EnsureBusinessAccounts/SandboxProvisioner) *(source: `cmd/server/main.go, cmd/billing-worker/main.go, namespace-001/pf-006a`)*
- Always self-register new billing backends via app.Service.RegisterMarketplaceListing in the app's New() constructor implementing billing.InvoicingApp — never hardcode provider logic inside billing.Service; register charge LineEngines only in app/common/charges.go *(source: `openmeter/app/stripe/service/factory.go, dep-005/billing-001`)*
- Always edit TypeSpec under api/spec/packages/aip (v3) or packages/legacy (v1) with @route/@tag only in the root openmeter.tsp/main.tsp, then run make gen-api AND make generate; adding @query/HTTP decorators requires importing @typespec/http with 'using TypeSpec.Http;' *(source: `api/spec/CLAUDE.md, Makefile, gen-003/gen-004/api-003`)*
