# .codex

<!-- archie:ai-start -->

> Tooling-only root configuring how this repo runs under OpenAI Codex. Splits into agents/ (Codex agent profiles, e.g. the archie_analysis subagent) and rules/ (execpolicy auto-approve rules for Archie's deep-scan / intent-layer / share / viewer scripts). Contains no application or build logic — both children are generated by the Archie Codex installer.

## Patterns

**Agent profiles vs execpolicy rules separation** — agents/ holds one TOML per agent profile (workspace-write sandbox, scoped writes); rules/ holds prefix_rule entries that auto-approve the exact shell commands Archie workflows invoke. The two concerns never mix. (`// .codex/agents/archie-analysis.toml configures the agent; .codex/rules/archie.rules auto-approves its commands`)
**Narrow per-script auto-approve** — Each Archie workflow script gets its own prefix_rule with an explicit match list rather than a blanket python3 allow; most-restrictive-wins precedence governs overlaps. (`prefix_rule { program = "python3", args = [...] } // one per .archie script`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `agents/archie-analysis.toml` | Codex subagent profile used by Archie scan/deep-scan/intent-layer workflows. | Do not widen sandbox_mode beyond workspace-write or remove the 'stay within your assigned task / do not paste large artifacts' developer_instructions. |
| `rules/archie.rules` | Generated execpolicy rules auto-approving Archie workflow commands. | Regenerated at install time — hand edits do not persist; a new .archie workflow script without a matching prefix_rule will prompt the user mid-run. |

## Anti-Patterns

- Adding application or build logic here — this folder only configures Codex agents and execpolicy.
- Widening sandbox_mode beyond workspace-write without intent — undermines the scoped-write safety guarantee.
- Granting broad allow rules (e.g. all python3) instead of script-scoped prefix_rules with explicit match lists.
- Hand-editing archie.rules and expecting it to persist — it is regenerated by the installer.

## Decisions

- **Archie ships a dedicated Codex worker agent rather than reusing the default agent.** — Bounds automated Archie runs with scoped writes and explicit instructions, keeping them safe and predictable.
