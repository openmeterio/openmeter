# internal

<!-- archie:ai-start -->

> Single Wire DI entry point for the jobs binary: provides the Wire-generated Application struct, Viper config loading, and the package-level globals (App, AppShutdown, Config, ConfigFile) shared by every cmd/jobs sub-command package. Import direction is one-way outward — never import this from app/common.

## Patterns

**Wire-generated initializeApplication in wire.go** — wire.go declares the Application struct and calls wire.Build with app/common provider sets; wire_gen.go is the generated output. After any change to wire.go or constructors run 'make generate'. (`wire.Build(common.App, common.Billing, common.Customer, ..., wire.Struct(new(Application), "*"))`)
**Application struct as exhaustive service registry** — Every domain service used by any sub-command is a typed field on Application; adding a dependency means a new field plus a matching common.* provider in wire.Build, then regenerate. (`type Application struct { common.GlobalInitializer; BillingRegistry common.BillingRegistry; EntitlementRegistry *registry.Entitlement; ... }`)
**Two-phase startup: loadConfig then initializeApplication** — app.go's InitializeApplication calls loadConfig (populating Config) first, then Wire-generated initializeApplication(ctx, Config), storing App and AppShutdown. Sub-commands invoke it from PersistentPreRunE. (`App, AppShutdown, err = initializeApplication(ctx, Config)`)
**Package-level globals (nolint:gochecknoglobals)** — App, AppShutdown, Config, ConfigFile are deliberate package-level globals consumed across all job sub-packages; the nolint annotation is intentional. (`//nolint:gochecknoglobals
var (App Application; AppShutdown func(); Config config.Configuration; ConfigFile string)`)
**metadata() stamps service name 'jobs'** — metadata() in wire.go maps config.Configuration to common.Metadata with service name 'jobs', stamping all telemetry. (`func metadata(conf config.Configuration) common.Metadata { return common.NewMetadata(conf, version, "jobs") }`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `wire.go` | Source of truth for which common.* provider sets are included for the jobs binary; declares Application struct fields. | Adding a domain service to Application requires the corresponding common.* provider in wire.Build then 'make generate'. |
| `wire_gen.go` | Generated by Wire — DO NOT EDIT. Shows exact construction order and cleanup chain. | Re-run 'make generate' after any wire.go change; never hand-edit. |
| `app.go` | Public InitializeApplication entrypoint called by cmd/jobs/main.go PersistentPreRunE. | Order matters: loadConfig runs before initializeApplication; config validation failure aborts before Wire wiring. |
| `config.go` | loadConfig loads and validates config.Configuration via Viper with config.DecodeHook() and SetViperDefaults. | viper.ConfigFileNotFoundError is tolerated (env-var-only config); all other read errors are fatal. Config.Validate() runs last. |
| `globals.go` | Defines the four package-level globals consumed by all job sub-packages. | Application fields must exist before any sub-command can reference them at compile time. |

## Anti-Patterns

- Editing wire_gen.go directly instead of modifying wire.go and regenerating
- Adding domain service construction logic in app.go or config.go instead of wire.go provider sets
- Importing cmd/jobs/internal from app/common (creates an import cycle — direction is one-way outward)
- Adding an Application field without a corresponding common.* provider in wire.Build
- Calling viper.SetDefault directly instead of relying on app/config.SetViperDefaults

## Decisions

- **All domain wiring delegated to app/common provider sets rather than inline construction** — Centralizing providers in app/common means all binaries share identical wiring; constructor-signature changes surface at Wire generation time, not runtime.
- **Package-level globals for App/Config rather than Cobra context values** — Cobra's PersistentPreRunE runs before any sub-command RunE, making globals the simplest cross-package handoff without threading context through every layer.

## Example: Add a new service to the jobs Application

```
// 1. In wire.go add a field and provider:
type Application struct {
  // ... existing fields
  Currencies currencies.Adapter
}
// In wire.Build add: common.Currencies,

// 2. Run: make generate  (wire_gen.go regenerated)

// 3. Sub-commands access: internal.App.Currencies
```
