# internal

<!-- archie:ai-start -->

> Provides the Wire-wired Application struct, config loading, and package-level globals (App, AppShutdown, Config) shared by all cmd/jobs sub-commands; it is the single DI entry point for the jobs binary.

## Patterns

**Wire-generated initializeApplication** — wire.go declares the Application struct and calls wire.Build with common.* provider sets; wire_gen.go is the generated output. Never edit wire_gen.go manually. (`wire.Build(common.App, common.Billing, common.Customer, ..., wire.Struct(new(Application), "*"))`)
**Application struct as the service registry** — Every domain service needed by any job sub-command is a field on Application. Adding a dependency requires adding it as a field here and regenerating wire_gen.go with 'make generate'. (`type Application struct { common.GlobalInitializer; BillingRegistry common.BillingRegistry; EntitlementRegistry *registry.Entitlement; ... }`)
**Two-phase startup: loadConfig then initializeApplication** — app.go calls loadConfig first to populate Config, then calls Wire-generated initializeApplication(ctx, Config) to build App. Sub-commands call internal.InitializeApplication from their PersistentPreRunE. (`App, AppShutdown, err = initializeApplication(ctx, Config)`)
**Package-level globals with nolint:gochecknoglobals** — App, AppShutdown, Config, ConfigFile are deliberately package-level globals annotated with //nolint:gochecknoglobals — do not change to local variables. (`var App Application; var AppShutdown func(); var Config config.Configuration`)
**metadata function maps config to common.Metadata** — The metadata() function in wire.go bridges config.Configuration to common.Metadata using the service name 'jobs'; this is how all binaries stamp their telemetry identity. (`func metadata(conf config.Configuration) common.Metadata { return common.NewMetadata(conf, version, "jobs") }`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `wire.go` | Declares Application struct and wire.Build injection; source of truth for which common.* provider sets are included for jobs binary. | Adding a new domain service to Application requires adding the corresponding common.* provider set in wire.Build, then running 'make generate'. |
| `wire_gen.go` | Generated by Wire — DO NOT EDIT. Shows the exact construction order and cleanup chain. | Never hand-edit; re-run 'make generate' after any wire.go change. |
| `globals.go` | Defines the four package-level globals consumed by all job sub-packages. | All sub-command packages read from internal.App — fields must exist before a sub-command can reference them. |
| `app.go` | Public entrypoint InitializeApplication called by cmd/jobs/main.go's PersistentPreRunE. | Order matters: loadConfig before initializeApplication; any config validation failure aborts before Wire wiring runs. |
| `config.go` | Loads and validates app/config.Configuration using Viper; uses config.DecodeHook() for custom type unmarshal. | ConfigFileNotFoundError is tolerated (env-var only config); other read errors are fatal. |

## Anti-Patterns

- Editing wire_gen.go directly instead of modifying wire.go and regenerating
- Adding domain service construction logic inside app.go or config.go instead of wire.go provider sets
- Importing cmd/jobs/internal from app/common (would create import cycle)
- Adding a new sub-command field to Application without a corresponding common.* provider in wire.Build

## Decisions

- **All domain wiring is delegated to app/common provider sets rather than inline construction.** — Centralizing providers in app/common means all seven binaries share the same wiring logic; changes to a constructor signature surface at Wire generation time, not at runtime.
- **Package-level globals for App/Config rather than passing via cobra context.** — Cobra's PersistentPreRunE runs before any sub-command RunE, making globals the simplest cross-package handoff without threading context through every layer.

## Example: Adding a new service (e.g. CurrenciesService) to the jobs Application

```
// 1. In wire.go — add field and provider:
type Application struct {
	// ... existing fields
	Currencies currencies.Adapter
}
// In wire.Build add: common.Currencies,

// 2. Run: make generate
// wire_gen.go is regenerated automatically.
```
