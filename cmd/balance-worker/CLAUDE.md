# balance-worker

<!-- archie:ai-start -->

> Binary entrypoint for the balance-worker: subscribes to Kafka system and ingest topics via Watermill, recalculates metered entitlement grant burn-down using the credit engine and ClickHouse usage queries, and dispatches balance threshold notifications. All business logic lives in openmeter/entitlement/balanceworker; this folder only wires and starts it.

## Patterns

**Wire-only entrypoint** — wire.go declares the Application struct and the wire.Build call; wire_gen.go is fully generated. main.go only parses config and calls initializeApplication, app.SetGlobals(), app.Migrate(ctx), app.Run() — no business logic. (`app, cleanup, err := initializeApplication(ctx, conf); defer cleanup(); app.SetGlobals(); app.Migrate(ctx); app.Run()`)
**Shared lifecycle embedding** — Application embeds common.GlobalInitializer, common.Migrator, and common.Runner — all lifecycle methods (SetGlobals, Migrate, Run) come from those shared structs, never defined locally. (`type Application struct { common.GlobalInitializer; common.Migrator; common.Runner; Logger *slog.Logger; RuntimeMetricsCollector common.RuntimeMetricsCollector }`)
**Config FieldsOf extraction** — Worker-specific sub-config slices are extracted via wire.FieldsOf before being passed to provider sets, so the composite sets receive precisely the config struct they expect. (`wire.FieldsOf(new(config.Configuration), "BalanceWorker"), wire.FieldsOf(new(config.BalanceWorkerConfiguration), "ConsumerConfiguration")`)
**Binary-local metadata function** — Each binary defines a local metadata() function passing version and the literal binary name to common.NewMetadata; the name string appears in telemetry labels. (`func metadata(conf config.Configuration) common.Metadata { return common.NewMetadata(conf, version, "balance-worker") }`)
**Version via ldflags + runtime/debug fallback** — version/revision/revisionDate are provisioned by ldflags at build time; init() falls back to debug.ReadBuildInfo() vcs settings when ldflags are absent. (`var version string; func init() { if version == "" { version = "unknown" } ; buildInfo, _ := debug.ReadBuildInfo() ... }`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `wire.go` | Declares the Application struct fields and all common.* provider sets passed to wire.Build. Edit here when adding a new service dependency. | Has build tag //go:build wireinject — never remove it. Adding a domain service field requires also adding its provider set to wire.Build. |
| `wire_gen.go` | Generated by `go generate` (Wire). Contains the full initializeApplication wiring sequence and cleanup chain. | Carries //go:build !wireinject and 'DO NOT EDIT' header — any manual edit is overwritten on the next `make generate`. |
| `main.go` | Parses pflag/viper config and invokes lifecycle in order: parse flags -> unmarshal -> Validate -> initializeApplication -> SetGlobals -> Migrate -> Run. | Do not add business logic here. app.Migrate(ctx) must precede app.Run(). ctx is context.Background() only because this is program root. |
| `version.go` | Holds ldflags-provisioned version and reads vcs.revision / vcs.time from build info in init(). | Defaults all three vars to 'unknown' when build info is missing; do not rely on them being populated in tests. |

## Anti-Patterns

- Adding business logic to main.go instead of openmeter/entitlement/balanceworker
- Manually editing wire_gen.go
- Removing the //go:build wireinject tag from wire.go
- Adding a new domain service dependency without adding its provider set to wire.Build
- Introducing context.Background() for anything other than the root ctx at program start

## Decisions

- **common.Runner encapsulates the oklog/run group; main.go never builds run.Group directly.** — Keeps shutdown ordering, signal handling, and goroutine lifecycle consistent across all workers without duplicating boilerplate.
- **BalanceWorker wires Notification, Svix, and Subject services even though it serves no HTTP.** — Balance threshold events are dispatched to notification.Service after burn-down recalculation, requiring the full Svix/notification plumbing in this binary.

## Example: Adding a new provider set to the balance-worker DI graph

```
// In wire.go, add to wire.Build:
common.MyNewDomain,
// Add the service field to Application:
MyNewService mynewdomain.Service
// Then run: make generate
```
