# balance-worker

<!-- archie:ai-start -->

> Binary entrypoint for the balance-worker: subscribes to Kafka system and ingest topics via Watermill, recalculates metered entitlement grant burn-down, and dispatches balance threshold notifications. All business logic lives in openmeter/entitlement/balanceworker; this folder only wires and runs it.

## Patterns

**Wire-only entrypoint** — wire.go declares the Application struct and wire.Build call; wire_gen.go is generated. main.go only calls initializeApplication, app.SetGlobals(), app.Migrate(), app.Run(). No business logic in main.go. (`app, cleanup, err := initializeApplication(ctx, conf); defer cleanup(); app.SetGlobals(); app.Run()`)
**Shared Application interface via embedding** — Application embeds common.GlobalInitializer, common.Migrator, and common.Runner — all lifecycle methods come from those shared structs, not defined locally. (`type Application struct { common.GlobalInitializer; common.Migrator; common.Runner; Logger *slog.Logger; ... }`)
**Config extraction via wire.FieldsOf** — Worker-specific sub-config slices are extracted via wire.FieldsOf before passing to provider sets: wire.FieldsOf(new(config.Configuration), "BalanceWorker") and wire.FieldsOf(new(config.BalanceWorkerConfiguration), "ConsumerConfiguration"). (`wire.FieldsOf(new(config.BalanceWorkerConfiguration), "ConsumerConfiguration")`)
**Metadata function per binary** — Each binary defines a local metadata() function passing version and the binary name string to common.NewMetadata. The name string appears in telemetry labels. (`func metadata(conf config.Configuration) common.Metadata { return common.NewMetadata(conf, version, "balance-worker") }`)
**Version via ldflags + runtime/debug fallback** — version/revision/revisionDate are provisioned by ldflags. init() falls back to debug.ReadBuildInfo() vcs settings when ldflags are absent. (`var version string; func init() { if version == "" { version = "unknown" } }`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `wire.go` | Declares Application struct fields and lists all common.* provider sets passed to wire.Build. Edit here when adding a new service dependency to the binary. | wire.go has build tag //go:build wireinject — never remove it. wire_gen.go has //go:build !wireinject — never edit it manually. |
| `wire_gen.go` | Generated by `go generate` (wire). Contains the full initializeApplication function. DO NOT EDIT. | Any manual edit will be overwritten on next `make generate`. |
| `main.go` | Parses pflag/viper config, calls initializeApplication, and invokes lifecycle methods. Calls app.Migrate(ctx) before app.Run(). | Do not add business logic here. Keep startup sequence: parse flags → unmarshal config → validate → initializeApplication → SetGlobals → Migrate → Run. |

## Anti-Patterns

- Adding business logic to main.go instead of openmeter/entitlement/balanceworker
- Manually editing wire_gen.go
- Removing //go:build wireinject tag from wire.go
- Adding a new domain service dependency without adding its provider set to wire.Build in wire.go
- Introducing context.Background() for anything other than the root ctx at program start

## Decisions

- **common.Runner encapsulates the oklog/run group; main.go never builds run.Group directly.** — Keeps shutdown ordering, signal handling, and goroutine lifecycle consistent across all workers without duplicating boilerplate.
- **BalanceWorker depends on Notification and Subject services even though it does not serve HTTP.** — Balance threshold events are dispatched to notification.Service after burn-down recalculation, requiring Svix plumbing wired here.

## Example: Adding a new provider set to the balance-worker DI graph

```
// In wire.go, add to wire.Build:
common.MyNewDomain,
// Add the service field to Application:
MyNewService mynewdomain.Service
// Then run: make generate
```
