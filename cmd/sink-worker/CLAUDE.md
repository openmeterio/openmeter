# sink-worker

<!-- archie:ai-start -->

> Binary entry point for the Kafka-to-ClickHouse sink worker: reads raw ingest CloudEvents from Kafka, deduplicates them, batch-inserts into ClickHouse via streaming.Connector, then publishes ingest notification events to Watermill so the balance-worker can recalculate entitlements. Contains only wiring and startup — all sink logic lives in openmeter/sink.

## Patterns

**Wire-generated DI via initializeApplication** — wire.go declares the Application struct and wire.Build call; wire_gen.go is generated and must never be hand-edited. Add new dependencies by adding a field to Application and a provider to wire.Build. (`wire.Build(common.Sink, common.ClickHouse, common.Streaming, ..., wire.Struct(new(Application), "*"))`)
**oklog/run group for actor lifecycle** — main.go adds exactly three actors to run.Group: app.Sink.Run/Close, TelemetryServer.ListenAndServe/Shutdown, and run.SignalHandler. New background goroutines must be added as group.Add pairs, not standalone goroutines. (`group.Add(func() error { return app.Sink.Run(ctx) }, func(err error) { _ = app.Sink.Close() })`)
**Viper + pflag config loading with conf.Validate()** — Config is loaded via viper.Unmarshal into config.Configuration then validated with conf.Validate() before Wire initialization. Never skip validation or read config after initializeApplication. (`v.Unmarshal(&conf); conf.Validate(); app, cleanup, err := initializeApplication(ctx, conf)`)
**Metadata via common.NewMetadata with binary name** — The local metadata() function calls common.NewMetadata(conf, version, "sink-worker") to tag telemetry and logs with the binary identity. Every cmd binary must provide this function. (`func metadata(conf config.Configuration) common.Metadata { return common.NewMetadata(conf, version, "sink-worker") }`)
**All providers sourced from app/common, not local** — wire.Build references only common.* provider sets and the local metadata/NewLogger functions. No domain logic or direct service construction belongs in wire.go. (`common.Sink, common.Streaming, common.ClickHouse, common.WatermillNoPublisher`)
**run.WithReverseShutdownOrder for graceful shutdown** — group.Run(run.WithReverseShutdownOrder()) ensures Sink shuts down before telemetry server, matching the add order. Do not call group.Run() without this option. (`err = group.Run(run.WithReverseShutdownOrder())`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `wire.go` | Declares Application struct and wire.Build provider list. This is the DI spec — only edited when adding/removing app-level dependencies. | Build tag `//go:build wireinject` must be present; omitting it causes duplicate symbol errors with wire_gen.go. Never add business logic here. |
| `wire_gen.go` | Generated by `go generate ./...` (Wire). Contains the actual initializeApplication implementation with ordered cleanup chain. | Never hand-edit. Cleanup functions are chained in reverse-construction order — manual edits break the cleanup chain silently. |
| `main.go` | Entry point: config loading, Wire init, run group assembly, signal handling. Contains no domain logic. | app.SetGlobals() must be called before any logging to initialize OTel globals. The cleanup defer must remain unconditional even when initializeApplication errors. |
| `version.go` | Provides version/revision/revisionDate globals populated by ldflags and debug.BuildInfo. Shared pattern across all cmd binaries. | The `//nolint:gochecknoglobals` and `//nolint:gochecknoinits` suppressions are intentional — do not remove them. |

## Anti-Patterns

- Adding business logic or domain service calls directly in main.go or wire.go — all logic belongs in openmeter/sink or app/common
- Hand-editing wire_gen.go — it is fully generated and any manual change will be overwritten by `go generate`
- Introducing a new goroutine outside of run.Group — bypasses graceful shutdown and signal handling
- Constructing domain services (e.g. sink.Sink) directly in wire.go instead of delegating to common.Sink provider set
- Calling context.Background() for the run context — the root context is created in main() and passed through; use it

## Decisions

- **WatermillNoPublisher is used instead of a full Watermill publisher** — The sink worker only publishes ingest flush events via a dedicated Kafka publisher (NewSinkWorkerPublisher); it does not need the full system-events Watermill router, so WatermillNoPublisher keeps the wiring minimal.
- **Application struct embeds common.GlobalInitializer** — GlobalInitializer.SetGlobals() initializes OTel global meter/tracer/propagator in one call, ensuring all downstream packages that rely on otel globals are consistently configured.
- **Sink and TelemetryServer are the only two long-running actors** — Sink worker has a single responsibility (Kafka → ClickHouse). The telemetry server is a mandatory sidecar for health/metrics. Keeping the run group minimal reduces shutdown complexity.

## Example: Adding a new background actor (e.g. a reconciler) to the sink worker

```
// In wire.go — add field to Application:
Reconciler *myreconciler.Reconciler

// Add provider to wire.Build:
common.MyReconciler,

// In main.go — register with run group:
group.Add(
    func() error { return app.Reconciler.Run(ctx) },
    func(err error) { _ = app.Reconciler.Close() },
)
// Then run: go generate ./cmd/sink-worker/... to regenerate wire_gen.go
```
