# cmd

<!-- archie:ai-start -->

> Organisational root for all deployable binary entrypoints (server, billing-worker, balance-worker, sink-worker, notification-service, jobs, benthos-collector). Each child is a thin Wire+main.go launcher that loads Viper config, calls a Wire-generated initializeApplication, runs migrations, and starts an oklog/run.Group; all business logic lives in openmeter/* or app/common.

## Patterns

**Wire-only entrypoint** — Each binary contains only wire.go (//go:build wireinject + wire.Build over app/common provider sets), wire_gen.go (generated), main.go, and version.go. No domain logic or shared state lives here. (`cmd/sink-worker/wire.go: wire.Build(common.NewSinkWorker, ...); wire_gen.go generated by `make generate`.`)
**Lifecycle via oklog/run.Group** — Long-running actors (HTTP server, Kafka consumer, telemetry server, sink loop) are added to run.Group via common.Runner or explicit run.Add; no goroutine is spawned outside run.Group so graceful shutdown is preserved. (`cmd/sink-worker: run.Add(app.TelemetryServer.ListenAndServe, shutdown); run.Add(app.Sink.Run, cancel).`)
**Post-migration provisioning before Run** — Provisioning (EnsureBusinessAccounts, SandboxProvisioner, initNamespace, default billing profile) runs after app.Migrate(ctx) but before the run.Group starts consuming events, each guarded by error handling + os.Exit(1). (`cmd/billing-worker/main.go: app.Migrate(ctx) -> EnsureBusinessAccounts -> SandboxProvisioner -> run.Group.Run().`)
**Ordered namespace handler registration (server only)** — cmd/server is the only binary that registers namespace.Handler implementations; Ledger then KafkaIngest must be registered via manager.RegisterHandler before initNamespace so the default namespace receives every handler's provisioning. (`cmd/server/main.go: register ledger handler, register kafkaIngest handler, then initNamespace(ctx, app.NamespaceManager).`)
**Binary identity via common.NewMetadata + ldflags version** — Each binary's version.go declares `var version = "unknown"` overridden by ldflags (runtime/debug fallback) and a metadata() function feeding OTel service.name and log fields. (`cmd/balance-worker/version.go + common.NewMetadata(version, "balance-worker").`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `cmd/*/wire.go` | Hand-written Wire provider graph for the binary (//go:build wireinject + wire.Build). | Never remove the //go:build wireinject tag; adding a domain service requires adding its app/common provider set here and regenerating. |
| `cmd/*/wire_gen.go` | Generated DI assembly produced by `go generate`/`make generate` from wire.go. | Manual edits are overwritten on next generate; regenerate after any wire.go or constructor signature change. |
| `cmd/server/main.go` | Registers namespace handlers, provisions default namespace + billing profile, builds router.Config from all ~40 services, runs a five-actor run.Group (API, telemetry, Kafka producer, NotificationEventHandler goroutine, termination checker). | Handler order is load-bearing (Ledger before KafkaIngest, both before initNamespace); uses common.WatermillNoPublisher because the server does not consume topics. |
| `cmd/billing-worker/main.go` | Provisions ledger business accounts and sandbox app, wires AppRegistry (Sandbox/Stripe/CustomInvoicing), then starts the Watermill consumer loop. | EnsureBusinessAccounts and SandboxProvisioner must complete before run.Group.Run(); skipping EnsureBusinessAccounts leaves the ledger without business accounts. |
| `cmd/notification-service/main.go` | Wire assembles services, then main.go manually constructs the Watermill Kafka subscriber and notification consumer and runs them in run.Group. | Keep subscriber construction in main.go so the consumer group name comes from config at startup; use app.EventPublisher.Marshaler() (not app.MessagePublisher) for deserialization. |
| `cmd/sink-worker/main.go` | Loads + Validate()s Viper config, Wire-inits, runs Sink + TelemetryServer actors with reverse shutdown order and unconditional cleanup defer. | All sink logic lives in openmeter/sink; never construct sink.Sink/streaming.Connector directly here; use root ctx not context.Background() in actors. |
| `cmd/jobs (main.go + internal)` | Cobra CLI: PersistentPreRunE wires the whole app once into internal.App package-level globals; sub-commands source services from there. | Never construct domain services inline in RunE; always cmd.Context() not context.Background(); nil-guard optional app services; ledger backfill intentionally bypasses Wire for concrete adapters. |
| `cmd/benthos-collector/main.go` | Thin launcher that blank-imports collector plugin packages then calls service.RunCLI; no Wire DI. | Register new collector plugins via blank import here; do not add Wire or domain service instantiation. |

## Anti-Patterns

- Adding business logic to main.go instead of openmeter/* or app/common.
- Manually editing wire_gen.go instead of regenerating via `make generate`.
- Registering a namespace.Handler after initNamespace in cmd/server, so the default namespace misses that handler's provisioning.
- Calling EnsureBusinessAccounts/SandboxProvisioner after Run(), or spawning a goroutine outside run.Group.
- Introducing context.Background()/context.TODO() anywhere other than the root context at program start in main().

## Decisions

- **All business logic lives in openmeter/* or app/common; cmd/* is wiring and lifecycle only.** — Keeps domain packages independently testable and reusable across binaries without importing cmd/*.
- **Each binary owns its own Wire provider graph in wire.go rather than sharing one graph.** — Different binaries need different provider subsets; separate graphs prevent unintended coupling and reduce binary size.
- **cmd/notification-service constructs its Watermill subscriber in main.go, not in Wire.** — The consumer group name must come from runtime config at startup, which is unavailable during Wire's compile-time graph resolution.
