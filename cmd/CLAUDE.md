# cmd

<!-- archie:ai-start -->

> Organisational root for all seven deployable binary entrypoints. Each child is a thin Wire+main.go launcher that delegates all business logic to openmeter/* domain packages; no domain logic or shared state lives here.

## Patterns

**Wire-only entrypoint** — Each cmd/* binary contains only wire.go (provider declarations), wire_gen.go (generated), and main.go (config load + run.Group). Business logic belongs in openmeter/* or app/common. (`cmd/server/wire.go declares wire.Build(...); cmd/server/main.go calls initializeApplication and delegates to common.Runner.`)
**Post-migration provisioning before Run** — Provisioning steps (EnsureBusinessAccounts, SandboxProvisioner, initNamespace) must execute after DB migration but before the run.Group starts consuming events. (`cmd/billing-worker/main.go calls EnsureBusinessAccounts then run.Group.Run(); cmd/server/main.go calls initNamespace before adding HTTP listener actor.`)
**Namespace handler registration before initNamespace** — In cmd/server, all namespace.Handler implementations (Ledger, KafkaIngest) must be registered via manager.RegisterHandler before initNamespace is called so the default namespace receives all handler provisioning. (`cmd/server/main.go registers ledger handler then kafkaIngest handler then calls initNamespace(ctx, app.NamespaceManager).`)
**common.Runner / oklog run.Group for actor lifecycle** — Long-running actors (HTTP server, Kafka consumer, telemetry server) are added to run.Group via common.Runner or manually with run.Add. No goroutine is spawned outside run.Group. (`cmd/sink-worker: run.Add(app.TelemetryServer.ListenAndServe, ...); run.Add(app.Sink.Run, ...)`)
**Metadata via common.NewMetadata with binary name** — Each binary calls common.NewMetadata(version, binaryName) to populate OTel service.name and log fields. Version is read from ldflags or runtime/debug fallback. (`cmd/balance-worker/version.go: var version = "unknown"; common.NewMetadata(version, "balance-worker")`)

## Key Files

| File | Role | Watch For |
|------|------|-----------|
| `cmd/*/wire.go` | Declares the Wire provider graph for the binary. Contains //go:build wireinject and wire.Build call. | Never remove //go:build wireinject. Adding a new domain service requires adding its provider set to wire.Build here. |
| `cmd/*/wire_gen.go` | Generated by `go generate ./...` from wire.go. Do not edit manually. | Any manual edit is overwritten on next `make generate`. Regenerate after every wire.go or constructor signature change. |
| `cmd/server/main.go` | Largest main.go: registers namespace handlers, provisions default namespace and billing profile, builds router.Config, assembles run.Group. | Namespace handler registration order is load-bearing. KafkaIngest must follow Ledger. Both must precede initNamespace. |
| `cmd/billing-worker/main.go` | Provisions ledger business accounts and sandbox app before starting the Watermill consumer loop. | EnsureBusinessAccounts and SandboxProvisioner must complete before run.Group.Run(). |
| `cmd/notification-service/main.go` | Constructs Watermill Kafka subscriber in main.go (not Wire) so consumer group name can come from config at startup. | Do not move subscriber construction into Wire. Use app.EventPublisher.Marshaler() for event deserialization, not app.MessagePublisher. |
| `cmd/jobs/internal/wire.go` | Wires the entire application once via PersistentPreRunE; sub-commands use internal.App package-level globals. | Do not construct domain services inline in RunE. Always use cmd.Context() — never context.Background(). |
| `cmd/benthos-collector/main.go` | Thin launcher: blank-imports plugin packages then calls service.RunCLI. No Wire DI. | New collector plugins must be added as blank imports here. Do not add Wire or domain service instantiation. |

## Anti-Patterns

- Adding business logic directly in main.go instead of openmeter/* domain packages
- Manually editing wire_gen.go — always regenerate via `make generate`
- Registering a namespace.Handler after initNamespace in cmd/server
- Constructing domain services (billing, charges, ledger) inline in cmd/jobs RunE instead of using internal.App
- Introducing context.Background() anywhere other than the root context at program start in main()

## Decisions

- **All business logic lives in openmeter/* or app/common; cmd/* is wiring and lifecycle only.** — Keeps domain packages independently testable and reusable across binaries without importing cmd/*.
- **Each binary has its own Wire provider graph in wire.go rather than sharing a single graph.** — Different binaries need different subsets of providers; separate graphs prevent unintended coupling and reduce binary size.
- **cmd/notification-service constructs its Watermill subscriber in main.go, not in Wire.** — Consumer group name must come from config at startup, which is not available during Wire's compile-time graph resolution.
